# TRAE Work 技能（Skills）

> 来源：https://docs.trae.cn/solo_skills
> 入库时间：2026-06-21T13:00:00+08:00
> 主题：TRAE Work 技能系统
> 标签：trae, work, skills, skill-md

---

## TL;DR

- 技能通过 SKILL.md 定义，为智能体提供可复用专业能力
- 动态按需加载，仅相关时才加载详细内容，减少 Token 消耗
- 技能 vs 规则：规则全量加载，技能按需加载
- 技能 vs MCP：技能描述"如何做"，MCP 提供"可调用工具"
- 项目技能存于 `.trae/skills/`，全局技能存于 `~/.trae-cn/skills`
- 创建方式：上传本地（zip/.skill）或从技能市场安装

## 核心概念

### 技能定义

一个技能可以被视为提供给智能体的一套"专业能力说明书"（类似用户手册或操作指南）。在执行任务时，智能体可以按需加载相应的技能，从而增强其对任务的理解与执行能力。

### 特点

| 特点 | 说明 |
|---|---|
| 结构化 | 一个技能对应一个 `SKILL.md` 文件，文件中以结构化的方式描述完成某一类任务所需的信息，例如：任务目标与适用场景；关键约束与注意事项；推荐流程或操作步骤；可选的脚本、模板或示例 |
| 动态按需加载 | 智能体不会在任务开始时一次性读取所有技能的完整内容。在执行任务前，智能体会先扫描所有技能的简要描述，仅当判断当前任务与某个技能高度相关时，才会加载该技能的详细内容。这种按需加载机制可以有效减少上下文中的 Token 消耗、避免无关信息干扰智能体的决策 |

### 使用场景

| 场景 | 说明 |
|---|---|
| 保证输出结果的一致性与规范性 | 需要智能体在不同时间、不同任务中，始终按照既定标准输出结果。例如，将统一设计规范、执行团队标准、保持品牌一致性或确保代码符合项目约定等要求封装为技能，从而将隐性的个人或团队标准转化为显式、可复用的专业能力，最终使输出结果更加稳定、可控 |
| 自动化重复性工作流 | 需要频繁执行相同或高度相似的多步骤任务。例如，对于测试流程、代码规范检查、常规数据分析等难以避免的日常工作，可以将既有的 SOP 封装为技能。一旦相关任务被触发，智能体即可自动按照定义好的流程执行，从而减少重复的指令输入，提升效率 |
| 总结与共享专业能力 | 总结个人经验或团队规范，并在更大范围内复用。例如，将技能在社区、交流群等公共平台进行分享，从而在不同的智能体、项目、团队间复用相同的技能 |

### 技能 vs 其他功能

| 对比维度 | 技能 | 规则 |
|---|---|---|
| 加载机制 | 按需加载，仅在实际需要被调用时才加载到上下文中 | 全量加载，一旦开启对话，所有规则都会被注入并持续占用上下文窗口 |
| Token 消耗 | 显著降低 | 持续占用 |

| 对比维度 | 技能 | MCP Server |
|---|---|---|
| 定位 | 向 AI 描述如何完成任务 | 向 AI 提供可以调用的工具 |
| 示例 | 约定测试工程结构、POM 设计规范、测试用例编写执行流程 | 通过 Playwright MCP Server 获得页面操作等自动化测试能力 |

### 技能类型

| 类型 | 说明 |
|---|---|
| 项目技能 | 在项目内创建的技能，仅在当前项目中生效 |
| 全局技能 | 在设置中心创建的技能，在所有项目中生效 |

### 技能的运行环境

| 环境类型 | 适用任务 | 适用客户端 |
|---|---|---|
| 本地 | 仅对本地任务生效 | TRAE Work 桌面版 |
| 云端 | 仅对云端任务（及从 GitHub 拉取的项目）生效 | TRAE Work 网页版、桌面版 |

## 技能的结构

一个技能中必须包含一个 `SKILL.md` 文件，还可以根据实际需求添加其他文件，例如可执行的脚本、可参考的模板和风格指南等。

```
skill-name/
├── SKILL.md          # （必须）智能体的核心指令
├── examples/         # （可选）输入/输出示例
│   ├── input.md
│   └── output.md
├── templates/        # （可选）可复用的模板
│   └── component.tsx
└── resources/        # （可选）参考文件、运行脚本或素材
    └── style-guide.md
```

### 技能所在目录

| 技能类型 | 目录路径 |
|---|---|
| 项目技能 | 项目所在路径下的 `.trae/skills/` 目录 |
| 全局技能（macOS/Linux） | 本地根目录 `~/.trae-cn/skills` |
| 全局技能（Windows） | 本地根目录 `%userprofile%/.trae-cn/skills` |

### SKILL.md 文件格式

```markdown
---
name: 技能名称
description: 简要描述这个技能的功能和使用场景
---

# 技能名称

## 描述
描述这个技能的作用。

## 使用场景
描述触发这个技能的条件。

## 指令
清晰的分步说明，告诉智能体具体怎么做。

## 示例 (可选)
输入/输出示例，展示预期效果。
```

> 关键：YAML frontmatter 中的 `name` 和 `description` 是必须的，`description` 是 AI 自动匹配的关键字段。

## 创建技能

### 方式一：上传本地技能

待上传的技能必须为包含根级 SKILL.md 文件的 zip 或 .skill 文件，且 SKILL.md 文件必须包含以 YAML 格式编写的技能名称和描述。

步骤：

1. 在左侧的任务栏中，点击顶部区域中的 **技能** → 进入技能管理中心
2. 点击右上角的 **上传技能** 按钮
3. 在 **上传技能** 弹窗中，上传本地技能文件，然后点击 **确认** 按钮
4. TRAE Work 会解析该技能中的文件并安装该技能。安装完成后，该技能将被添加至 **已安装** 页签，并默认被启用

### 方式二：从技能市场安装技能

技能市场为不同使用场景提供了多种技能，涵盖数据分析、界面设计、开发工具等领域。

步骤：

1. 在左侧的任务栏中，点击顶部区域中的 **技能** → 进入技能管理中心
2. 在 **技能市场** 页签中，搜索所需技能，然后点击技能卡片右侧的 **+** 按钮
3. TRAE Work 将安装该技能。安装完成后，该技能将被添加至 **已安装** 页签，并默认被启用

> 提示：点击某个技能的卡片后，界面上会弹出该技能的详情窗口，可阅读技能说明以判断是否符合需求。

## 在对话中使用技能

### 方式一：手动调用技能

| 调用方式 | 说明 |
|---|---|
| `/` 命令 | 在对话框中输入 `/`，然后从列表中选取所需的技能 |
| 直接指令 | 直接向 AI 发送指令，明确告知 AI 调用某个技能来完成任务，例如："用 codemap 技能总结一下这个分支的改动" |
| 运行按钮 | 在技能管理中心的 **已安装** 页签中，点击某个技能的卡片以打开其详情窗口，然后点击窗口右下角的 **运行技能** 按钮 |

### 方式二：由 AI 自动调用技能

AI 会结合当前任务内容与各技能中定义的"适用场景"或"何时使用"之类的描述，判断是否需要加载某个技能，并在合适的阶段自动调用相关技能。

示例：假设你定义了一个代码审核技能，其触发条件为"当用户请求代码反馈或评审时"。当你提出"这个函数写得如何"、"帮我审核以下代码"之类的请求时，AI 会识别到这类请求与代码审核技能的相关性，并在无需额外指令的情况下加载并使用该技能，以完成对应的代码分析与反馈。

## 启用、禁用、卸载技能

| 操作 | 方法 |
|---|---|
| 启用、禁用技能 | 在技能管理中心的 **已安装** 页签中，找到目标技能，然后滑动卡片右侧的滑块来启用、禁用该技能 |
| 卸载技能 | 点击目标技能的卡片，然后点击详情弹窗左下角的 **卸载** 按钮 |

## 最佳实践

技能的开发非一蹴而就，而是一个持续迭代的过程。参考：[如何写好一个 Skill：从创建到迭代的最佳实践](https://docs.trae.cn/ide/best-practice-for-how-to-write-a-good-skill)

## 关键坑点

- 技能 `description` 不精确会导致 AI 无法自动匹配
- 项目技能和全局技能的目录不同，放错位置会导致不生效
- 云端环境只能使用云端技能，本地环境只能使用本地技能
- 上传本地技能必须是 zip 或 .skill 文件，且必须包含根级 SKILL.md
- SKILL.md 的 YAML frontmatter 中 name 和 description 为必须字段

## 可迁移到 AOS 的规则

- SKILL.md 格式可直接复用为 AOS Skill 标准
- 按需加载机制减少 Token 消耗，适用于 Agent 系统
- 技能的 `description` 字段设计是自动匹配的关键
- 技能结构可包含脚本/模板/示例，不仅是纯文本指令
