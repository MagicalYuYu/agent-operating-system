❌ 痛点

AI 助手很聪明，但它"没有记忆"

如果你经常用 AI 助手工作，这些场景一定不陌生——

🔄
重复介绍，浪费 30%+ 时间
每次新对话都要重新说明项目背景、技术栈、约束条件。AI 的"记忆力"仅限于当前对话窗口。
🔀
多项目上下文混淆
同时推进多个项目时，AI 经常把 A 项目的配置写到 B 项目里，张冠李戴，越帮越忙。
🕳️
同一个坑踩两次
上次犯过的错误，下次对话还会再犯。AI 没有机制记住"上次踩过的坑"和"为什么那样不行"。
📦
经验无法沉淀和复用
好不容易摸索出的最佳实践、踩坑经验，随着对话关闭而消失。下次又得从头来过。
🔇
对话一关，一切归零
AI 之前做的所有事情、所有决策，对话关闭后全部消失，无法延续到下次。

🔄 闭环

AOS 的标准工作流：从启动到归档，全程留痕

不只是"能跑"，而是把 AI 助手的工作过程变成可追溯、可审计、可复用的闭环——

🚀
Step 1
项目启动前：自动生成标准化文件结构 自动初始化
新建项目的瞬间，AOS 自动按标准模板创建专属目录与配套文件——项目配置、状态跟踪、知识引用一应俱全，无需手动建文件夹、写样板。
01_PROJECTS/{name}/ AGENTS.md README.md STATUS.md reference_links.md
无 AOS 手动建目录、写样板、遗漏配置，每个项目都从空白开始，重复造轮子
⚡
Step 2
工作过程：实时留存记录与上下文 实时落盘
每次会话开始自动加载历史记忆，结束时把新增经验、反馈、项目状态写回磁盘——不依赖 AI 运行时记忆，新对话从文件恢复完整上下文。
04_MEMORY/INDEX.md user/user_profile.md project/proj_{name}.md feedback/*.md 06_LOGS/{name}/{date}.log
无 AOS 上下文仅存于对话窗口，关闭即失忆，新对话要重新介绍项目背景、技术栈、约束
📦
Step 3
工作结束：自动归档规范操作 归档规范
交付物按"项目名 + 交付类型"分层归档，状态同步更新到 STATUS.md 和 04_MEMORY/project/——下次接手时无需"考古"，所有进度一眼可见。
07_EXPORTS/{name}/{deliverable}/ STATUS.md → 同步更新 proj_{name}.md → 同步更新 SKILL_REGISTRY.md → call_count+1
无 AOS 交付物散落各处，无分层归档，下次接手靠"考古"翻聊天记录，进度不可追溯
🔍
Step 4
每日自主巡检：版本与结构健康验证 自检脚本
运行自检脚本扫描版本号同步、交叉引用完整性、目录合规性——发现残留旧版本号或断链立即报警，杜绝"改了一处忘了十处"的隐患。
aos_check.py 版本号同步 交叉引用验证 目录合规扫描 SYSTEM_STATE.md
无 AOS 改了一处忘了十处，版本号不一致、断链无人发现，bug 潜伏到用户报障才暴露
🗂️
Step 5
文件结构规整：10 目录各司其职 强约束目录
每个目录有且仅有一个职责，禁止业务污染——系统规则、项目数据、临时缓存、知识库、日志各有归处，AOS 高效规整混乱的文件结构，AI 不再"找不着文件"。
00_BOOT 系统规则 01_PROJECTS 项目隔离 04_MEMORY 状态持久化 07_EXPORTS 成果输出 09_REFERENCE 唯一知识库
无 AOS 文件散落无规整，靠 find 和记忆找文件，AI"找不着北"，越用越乱直至无法维护

✓ 工作过程实时写回 04_MEMORY/feedback/，踩过的坑只踩一次，经验持续积累

🛡️ 能力

三大核心能力

解决 AI 助手最根本的三个问题——

🧠
跨会话记忆
→ 助手的笔记本
AI 每次对话自动读写记忆文件，新对话从磁盘恢复全部上下文。不再"失忆"，经验持续积累。
📁
项目隔离
→ 独立办公桌
每个项目有独立的配置、状态、文档。切换项目时自动加载对应上下文，杜绝"张冠李戴"。
🔧
技能复用
→ 标准操作手册
把常用工作流封装成 Skill，一次定义、反复调用。团队共享最佳实践，避免重复造轮子。

🚀 快速开始

四步上手 AOS

纯文件系统，无需安装任何依赖——

Step 1
获取 AOS
克隆仓库或下载压缩包，AOS 是纯文件结构，无需安装任何依赖。
git clone https://github.com/MagicalYuYu/agent-operating-system.git
Step 2
启动 AOS Viewer 桌面（推荐）
从 GitHub Release 下载 AOS Viewer.exe，双击即可启动——无需 Python 环境，自带 Liquid Glass 桌面界面，自动扫描文件系统展示全貌。
Step 3
在 AI 工具中打开
用 TRAE Work（推荐）、Claude Code 或 Codex 打开 AOS 目录，AI 会自动读取根目录的 AGENTS.md。
Step 4
配置 Rules 并验证
添加 3 条 Rules（详见配置指南），然后运行自检脚本验证安装。
python 03_TOOLS/scripts/aos_check.py
