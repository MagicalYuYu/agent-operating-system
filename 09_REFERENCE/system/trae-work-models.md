## 自定义模型

### 使用限制

- 仅 TRAE Work 桌面版支持添加自定义模型
- 添加的自定义模型仅支持在本地环境中使用

### 模型来源

| 来源 | 说明 |
|---|---|
| 模型服务商 | 从 TRAE 预设的模型服务商和模型列表中选择，按界面提示填写 API 密钥即可添加 |
| 自定义配置 | 通过配置 API 格式、请求地址、模型 ID 和鉴权信息，接入 TRAE 未预设的模型 |

### 添加预设模型服务商的模型（完整步骤）

1. 前往 **设置** > **模型**，进入模型管理面板
2. 点击 **添加模型** 按钮，打开 **添加模型** 窗口
3. 在预设服务商列表中，选择服务商
4. (部分模型) 选择计费方式：
   - **按量计费**：根据实际消耗的 Token 数量和模型刊例价计费
   - **Coding Plan**：订阅式大模型服务套餐包，专注于编码场景
   - **Agent Plan**：订阅式大模型服务套餐包，与 Coding Plan 相比，此套餐提供更全面的能力
5. 选择模型，支持以下方式：
   - 直接从列表中选择 TRAE 为每个服务商预置的模型（均为默认版本）
   - 若希望使用其他模型或特定版本，点击列表中的 **使用其他模型**，然后在输入框中填写模型 ID
6. 填写该模型鉴权所需的 API 密钥
7. (可选) 展开 **高级配置** 表单，按需设置参数（见下表）
8. 点击 **提交** 按钮，TRAE Work 将调用服务商接口检测 API 密钥有效性：
   - 连接成功 → 模型被添加到模型列表
   - 连接失败 → 窗口展示错误信息和服务商返回的日志，可根据这些信息排查问题

#### 预设服务商模型 — 高级配置参数

| 参数 | 描述 |
|---|---|
| 模型展示名称 | 设置模型在列表中的展示名称。若不设置，将默认展示模型 ID |
| 上下文窗口-输入 | 设置该模型单次请求可接收的最大 Token 数 |
| 上下文窗口-输出 | 设置该模型单次响应可生成的最大 Token 数 |
| 工具调用轮次 | 设置该模型在单次任务中可调用工具的最大次数 |
| 支持多模态 | 设置选择该模型时，是否允许用户提供图片作为输入。若选择 **不支持**，对话输入框左下角将不再显示 **添加图片** 图标，使用该模型时将无法上传图片作为输入 |

### 通过自定义配置添加模型（完整步骤）

1. 前往 **设置** > **模型**，进入模型管理面板
2. 点击 **添加模型** 按钮，打开 **添加模型** 窗口
3. 滚动至列表底部，然后选择 **自定义配置**
4. 设置模型的 API 格式
5. 填写自定义请求地址，支持两种方式：
   - **方式一**：打开 **完整 URL** 开关，然后填写完整地址
   - **方式二**：关闭 **完整 URL** 开关，然后填写基础地址
6. 输入模型 ID
7. 设置 **多模态** 开关（默认开启；关闭后对话输入框左下角不再显示 **添加图片** 图标，无法上传图片）
8. 填写该模型鉴权所需的 API 密钥
9. (可选) 展开 **高级配置** 列表，按需设置参数（见下表）
10. 点击 **添加模型** 按钮，TRAE 将调用服务商接口检测 API 密钥有效性：
    - 连接成功 → 模型被添加到模型列表
    - 连接失败 → 窗口展示错误信息和服务商返回的日志，可根据这些信息排查问题

#### 自定义配置模型 — 高级配置参数

| 参数 | 描述 |
|---|---|
| 模型系列 | TRAE 为特定模型系列优化了 Prompt 和超参，未选择时使用默认配置 |
| 模型展示名称 | 设置模型在列表中的展示名称。若不设置，将默认展示模型 ID |
| 上下文窗口 | 设置单次请求可接收的最大 Token 数，以及单次响应可生成的最大 Token 数 |
| 工具调用轮次 | 设置模型在单次任务中可调用工具的最大次数 |

## 关键坑点

- 自定义模型仅桌面版支持，仅本地环境可用
- 部分模型需选择计费方式（按量/Coding Plan/Agent Plan）
- 连接失败时需查看服务商返回的日志排查
- 预设服务商的高级配置有"上下文窗口-输入/输出"分开设置，自定义配置的"上下文窗口"是合并设置
- 自定义配置额外有"模型系列"参数，预设服务商无此参数

## 可迁移到 AOS 的规则

- 模型选择应根据任务类型（Work/Code）自动匹配
- 自定义模型接入需验证 API 密钥有效性
- 工具调用轮次配置直接影响 Agent 执行能力
- 多模态开关影响用户交互界面（是否显示添加图片按钮）
