## 版本亮点

v1.1.0 是 AOS 的第二个正式版本，本次更新的核心是 **AOS Viewer 可视化界面**——让你的 AI 操作系统有了自己的"控制台"。

---

## 新增功能

### 🖥 AOS Viewer 可视化界面

AOS 自带一个 Liquid Glass 玻璃拟态风格的仪表盘，让你一眼看清系统全貌：

- **只读仪表盘**：系统总览 / Skill 管理 / 项目工作区 / 记忆中心 / 知识库 / Loop 监控 / Agent 任务池 / 日志查看器
- **控制台**：一键触发自检 / 同步 / 迁移采集等内置脚本
- **三种启动方式**：
  - 打包 EXE（双击即可运行，无需安装 Python）
  - 桌面窗口模式（PyWebView）
  - 浏览器模式（直接打开 index.html）
- **i18n 国际化**：内置 zh-CN / en-US 双语切换
- **Spotlight 全局搜索**：搜索模块 / Skill / 项目 / 记忆 / 知识，含关键词高亮
- **菜单栏 + 下拉菜单**：文件 / 视图 / 工具 / 帮助 四大菜单
- **三种主题**：深色 / 浅色 / 极简
- **启动动画**：Boot Screen + 动态背景光晕

### 📊 数据生成脚本

新增 `aos_generate_data.py`，扫描 AOS 文件系统自动生成 data.js：

- 解析 `00_BOOT/SYSTEM_STATE.md` → 组件状态、运行时统计、事件日志
- 解析 `00_BOOT/SKILL_REGISTRY.md` → Skill 列表
- 扫描 `01_PROJECTS/` → 项目列表（AGENTS.md + STATUS.md + README.md）
- 扫描 `04_MEMORY/` → 记忆索引和内容
- 扫描 `09_REFERENCE/` → 知识库索引和内容

### 🔄 GitHub 同步脚本

新增 `aos_sync_github.py`，安全地将 AOS 框架同步到 GitHub 上传目录：

- **白名单模式**：默认不上传，只允许特定文件
- **内容脱敏**：自动替换敏感信息（凭据 / IP / Token / 个人昵称等）
- **安全验证**：同步后自动执行安全检查，确认无敏感文件残留

### 📋 巡检规范

新增系统级 + 项目级双层巡检规范：

- `00_BOOT/SYSTEM_INSPECTION.md` — 系统级巡检（独立专用会话执行）
- `00_BOOT/PROJECT_INSPECTION.md` — 项目级巡检（项目工作会话前置）
- 四类文件分类标准（A 冗余 / B 过时 / C 留存 / D 迁移）
- 统一的路径层级规范（项目名 → 模块名 → 类型 → 日期）

## 改进

### 项目类型识别

- 新增"项目集"识别：AGENTS.md 含"子项目清单" section 即自动判定为项目集
- 支持插件集、微服务集等多种项目集形态

### 项目状态解析

- 重写状态解析逻辑，支持 4 级优先级（元信息状态 → 当前阶段 → 阶段统计 → 全文搜索）
- 支持中文状态标记（✅完成 / 🔄进行中 / ⏳待办）和无 emoji 格式
- 支持英文状态标记（completed / pending / in_progress）

### 项目描述解析

- 跳过标题 / 引用 / 分割线 / 徽章 / 图片 / 目录 / 表格 / 代码块行
- 取第一个完整段落，截断到 300 字符

### 示例项目

新增 3 个精心制作的示例项目，演示不同项目类型的标准结构：

- `_example_cli_tool` — 单一项目（Python 日志分析工具，零外部依赖）
- `_example_game_localization` — 单一项目（游戏本地化，翻译文件 + 完整流程文档）
- `_example_plugin_suite` — 项目集（聊天机器人插件集合，含 3 个独立插件 + EventBus 通信）

### 文档体系

- 所有目录 README 统一更新为 v1.1.0
- 新增 5 个目录 README：01_PROJECTS / 05_CACHE / 06_LOGS / 08_INBOX / 99_ARCHIVE
- AOS Viewer 使用说明完善，补充打包 EXE 启动方式
- 跨平台适配说明（TRAE / Claude Code / Codex）

### AOS_Viewer.exe 使用说明

1. 下载 `AOS_Viewer.exe`
2. 放到 AOS 根目录（与 `AGENTS.md` 同级）
3. 双击运行，自动打开桌面窗口
4. 启动后自动扫描 AOS 文件系统生成实时数据

> 如未放在 AOS 根目录，将显示内置示例数据。

## 升级指南

### 从 v1.0.0 升级

1. 备份现有 AOS 目录（特别是 `04_MEMORY/` 和 `01_PROJECTS/`）
2. 下载 v1.1.0 源码或 git pull
3. 将备份的 `04_MEMORY/` 和 `01_PROJECTS/` 恢复到新版本对应位置
4. 运行 `python 03_TOOLS/scripts/aos_check.py` 验证一致性

### 配置 Rules

如使用 TRAE，添加 3 条 Rules（详见 [配置指南](../docs/trae-setup-guide.md)）：

- `aos-consistency-guard` — 修改文件时触发一致性检查
- `aos-memory-rules` — 读写 04_MEMORY/ 时触发记忆规则
- `aos-skill-rules` — 读写 03_TOOLS/skills/ 时触发 Skill 规则
