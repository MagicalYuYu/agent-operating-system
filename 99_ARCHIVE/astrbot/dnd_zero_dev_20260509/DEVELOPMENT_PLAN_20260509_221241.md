## 一、项目愿景与定位

### 1.1 项目目标

开发一款基于D&D 5e规则的二次元手游风格TRPG跑团插件，专为QQ群聊场景设计，融合LLM智能剧情生成与长线养成要素。

### 1.2 核心定位 ⚠️ 重要

**游戏本质认知**：

本插件本质属于**二次元网络手游**，而非传统TRPG工具。玩家日常默认处于**主菜单界面（城镇）**，可访问公会、商店等各类功能菜单。

**界面层级架构**：
```
┌─────────────────────────────────────────────────────────────┐
│                    游戏主界面（城镇）                          │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐           │
│  │ 📜 副本 │ │ 🏠 公会 │ │ 🛒 商店 │ │ 👤 角色 │           │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘           │
│                                                             │
│  玩家可在此进行：角色管理、查看状态、访问商店、与NPC交互等       │
│  ⚔️ 战斗场景仅在参与副本或特定事件时触发                        │
└─────────────────────────────────────────────────────────────┘
                           ↓ 进入副本
┌─────────────────────────────────────────────────────────────┐
│                    副本场景（战斗/探索）                       │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐           │
│  │ ⚔️ 战斗 │ │ 🔍 探索 │ │ 🧩 解谜 │ │ ❓ 奇遇 │           │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘           │
│                                                             │
│  在副本内可触发：回合制战斗、地图探索、谜题机关、随机事件        │
└─────────────────────────────────────────────────────────────┘
```

**核心特点**：
- **菜单化交互**：玩家通过指令菜单进行各类操作
- **城镇为家**：日常活动在主界面进行
- **副本为战场**：战斗/探索等内容在副本内进行
- **LLM赋能**：副本内容由大语言模型动态生成

### 1.3 设计理念

1. **二次元手游体验**：类游戏主菜单界面，清晰的功能分区
2. **沉浸式剧情**：通过LLM生成丰富的场景描写和事件
3. **规则透明**：所有D&D规则判定过程清晰展示
4. **长线养成**：角色成长、装备收集、资源积累
5. **社交协作**：组队副本、多人互动

### 1.4 目标用户

- TRPG新手玩家
- 二次元游戏爱好者
- 喜欢角色扮演的QQ群用户
- 追求长线养成和社交的玩家

## 二、游戏架构设计 ⚠️

### 2.1 界面层级架构

本插件采用**三层界面架构**，玩家在不同状态下处于不同界面：

```
┌─────────────────────────────────────────────────────────────┐
│                    第一层：主界面（城镇大厅）                     │
├─────────────────────────────────────────────────────────────┤
│  玩家状态：日常休闲、无战斗压力                                  │
│  可访问功能：                                                  │
│  • 角色管理（查看/编辑角色卡）                                 │
│  • 副本入口（浏览/选择副本）                                    │
│  • 商店系统（购买/出售物品）                                    │
│  • 公会功能（查看排名/成员）                                    │
│  • 个人背包（查看装备/道具）                                    │
│  • 签到系统（每日奖励）                                        │
│                                                              │
│  ⚠️ 此界面下攻击指令受限，仅能使用无害互动                        │
└─────────────────────────────────────────────────────────────┘
                              ↓ 选择进入副本
┌─────────────────────────────────────────────────────────────┐
│                    第二层：副本界面（冒险场景）                     │
├─────────────────────────────────────────────────────────────┤
│  玩家状态：副本进行中、可触发战斗                                 │
│  副本类型：                                                   │
│  • 🌲 森林秘境 - 初级副本，侧重探索                              │
│  • 🏚️ 地下墓穴 - 中级副本，侧重解谜                            │
│  • ⚔️ 战场遗迹 - 高级副本，侧重战斗                            │
│  • 🌟 龙穴探险 - 终极副本，综合挑战                             │
│                                                              │
│  副本环节：奇遇 → 探索 → 解谜 → 战斗 → 奖励                     │
└─────────────────────────────────────────────────────────────┘
                              ↓ 遭遇敌人
┌─────────────────────────────────────────────────────────────┐
│                    第三层：战斗界面（回合制战斗）                   │
├─────────────────────────────────────────────────────────────┤
│  玩家状态：战斗中、严格遵循D&D规则                               │
│  战斗机制：                                                   │
│  • 先攻排序（按敏捷决定行动顺序）                                 │
│  • 回合动作（攻击/防御/移动/技能）                               │
│  • 生命值管理（受伤/治疗）                                      │
│  • 状态效果（增益/减益）                                        │
│  • 战斗结束（奖励结算）                                         │
│                                                              │
│  ⚠️ 战斗必须由副本机制触发，禁止在城镇随意PK                       │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 状态机设计

```
┌─────────────────────────────────────────────────────────────┐
│                      游戏状态机                                │
└─────────────────────────────────────────────────────────────┘

    ┌──────────┐     进入副本      ┌──────────┐     遭遇敌人    ┌──────────┐
    │          │ ───────────────→ │          │ ────────────→ │          │
    │  城镇    │                  │  副本    │                │  战斗    │
    │  状态    │ ←─────────────── │  状态    │ ←─────────── │  状态    │
    │          │     退出副本      │          │    战斗结束   │          │
    └──────────┘                  └──────────┘                └──────────┘
         ↑                            ↑                            ↓
         │                            │                            │
         └────────────────────────────┴────────────────────────────┘
                                    │
                              事件/指令触发
```

**状态转换规则**：

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| 城镇 | 进入副本指令 | 副本 | 玩家选择副本并进入 |
| 副本 | 遭遇敌人事件 | 战斗 | 自动触发，进入回合制 |
| 战斗 | 敌人全灭/逃跑 | 副本 | 继续副本探索 |
| 副本 | 通关/退出指令 | 城镇 | 副本结束，返回城镇 |
| 战斗 | 全灭/逃跑失败 | 城镇 | 副本失败，返回城镇 |

### 2.4 副本内容体系

副本是核心玩法载体，包含多种内容环节：

#### 2.4.1 副本环节类型

| 环节类型 | 说明 | 典型场景 | 机制 |
|----------|------|----------|------|
| **奇遇** | 随机触发的特殊事件 | 发现宝箱、遇见NPC、触发陷阱 | 概率触发，自动判定 |
| **探索** | 地图探索与资源收集 | 搜索房间、发现线索、收集材料 | 技能检定+LLM描写 |
| **解谜** | 谜题机关与逻辑挑战 | 密码门、机关阵、谜语 | 智力检定+提示系统 |
| **战斗** | 回合制敌人遭遇 | 遭遇怪物、BOSS战、守卫战 | D&D规则战斗 |
| **剧情** | 剧情对话与选择 | NPC对话、道德选择、任务交接 | LLM生成+分支选择 |
| **奖励** | 战斗/通关结算 | 经验、金币、装备、道具 | 随机+等级平衡 |

## 三、核心原则与约束

### 3.1 新手友好原则 ⚠️ 重要

**每个数值必须说明来源**：
- ❌ 旧: `D20: **8** + +2` → 不知道+2从哪来
- ✅ 新: `D20: **8** + 敏捷调整(+2) = 10`

**展示格式规范**：
- 每行最多显示2-3个技能（列表）
- 表格宽度不超过30字符
- 使用emoji辅助理解
- 关键数值加粗显示

### 3.2 中英文双语指令原则 ⚠️ 重要

**所有指令必须同时支持**：
- `/dnd roll` = `/dnd 掷骰`
- `/dnd skill` = `/dnd 技能`
- `/dnd attack` = `/dnd 攻击`

**注册方式**：
```python
@dnd.command("roll")
@dnd.command("掷骰")  # 中文别名
```

### 3.3 功能互斥原则 ⚠️ 重要

**战斗/副本完成后限制随意攻击**：
- 进入副本后，只能对副本内怪物发起攻击
- 战斗模式开启后，攻击指令只能在战斗中使用
- 普通模式下，攻击指令需指定目标且限制频率

### 3.4 数据持久化原则

- 所有玩家数据保存到 `plugin_data/astrbot_plugin_dnd_zero/data/`
- 每次修改前自动备份到 `backup/` 目录
- 文件命名规范: `原文件名_YYYYMMDD_HHMMSS.扩展名`

### 3.5 文档同步原则 ⚠️ 重要

**每次更新必须同步更新**：
- [ ] `README.md` - 项目说明
- [ ] `USER_GUIDE.md` - 用户使用指南
- [ ] `CHANGELOG.md` - 变更日志
- [ ] `backup/` - 自动备份

```python
async def dnd_attack(self, event, target: str):
    # 检查是否在战斗/副本中

        if not self._check_attack_cooldown(event):
```

## 六、v0.3.0 LLM副本核心玩法 ⚠️ 核心重点

### 6.1 设计理念 ⚠️

**LLM副本是本插件的核心差异化竞争力**

传统TRPG插件只能提供基础的骰点功能，玩家需要自行脑补剧情。本插件通过LLM大语言模型实现：
- 智能剧情生成
- 动态事件判定
- NPC对话交互
- 场景描写渲染
- 规则辅助判定

**设计原则**：
1. **沉浸感优先**：通过丰富的文本描写让玩家身临其境
2. **规则透明**：所有判定规则清晰展示，不黑箱
3. **可干预性**：玩家可随时介入剧情走向
4. **容错机制**：LLM生成内容需经过规则校验

#### 6.6.3 规则校验器规则

```python
class RuleVerifier:
    @staticmethod
    def verify_damage(damage: int, max_allowed: int) -> bool:
        """验证伤害值不超过合理范围"""
        return damage <= max_allowed

    @staticmethod
    def verify_check_result(check_type: str, result: int, dc: int) -> bool:
        """验证检定结果逻辑正确"""
        return True  # 实际使用DiceRoller结果

    @staticmethod
    def verify_loot(rewards: List[Item], party_level: int) -> bool:
        """验证战利品价值与等级匹配"""
        total_value = sum(item.value for item in rewards)
        max_value = party_level * 100  # 每级最多100金币价值
        return total_value <= max_value

    @staticmethod
    def verify_npc_scaling(npc: NPC, party_level: int) -> bool:
        """验证NPC强度与队伍等级匹配"""
        level_diff = abs(npc.level - party_level)
        return level_diff <= 3  # 等级差距不超过3级
```

## 十、副本内容设计 ⚠️ 核心玩法

### 10.1 副本环节详解

副本是玩家的主要活动场所，包含六大核心环节：

#### 10.1.1 奇遇环节 🆕

**定义**：随机触发的特殊事件，为副本增添变数和惊喜。

**触发机制**：
- 每次进入新场景有20-30%概率触发
- 某些特定地点必触发（如：宝箱房间）
- 连续探索失败增加触发概率

**奇遇类型**：

| 类型 | 效果 | 示例 |
|------|------|------|
| 宝箱发现 | 获得额外奖励 | 发现隐藏宝箱，获得金币+100 |
| NPC偶遇 | 获得任务/情报 | 遇见流浪商人，可购买稀有道具 |
| 陷阱触发 | 遭受伤害/负面效果 | 踩到陷阱，受到3d6伤害 |
| 随机增益 | 获得临时Buff | 发现魔法泉水，HP全满+临时+2AC |
| 隐藏剧情 | 触发支线故事 | 发现前任冒险者遗书，获得隐藏任务 |

**实现代码示例**：
```python
class EncounterEvent:
    """奇遇事件"""
    event_type: str
    trigger_chance: float
    description: str
    effects: List[Effect]
    requires_check: bool
    skill_type: Optional[str]  # 如需要检定

async def check_encounter(scene: Scene, player: Character) -> Optional[EncounterEvent]:
    """检查是否触发奇遇"""
    roll = DiceRoller.roll_d100()
    if roll <= scene.encounter_chance:
        return generate_encounter(scene.difficulty, player.level)
    return None
```

#### 10.1.2 探索环节 🔍

**定义**：在副本中移动、搜索、发现线索和资源。

**探索动作**：

| 动作 | 检定类型 | 效果 |
|------|----------|------|
| 搜索房间 | 调查/感知 | 发现隐藏物品或线索 |
| 观察环境 | 感知 | 发现陷阱或隐藏敌人 |
| 探索路径 | 生存/智力 | 选择最佳前进路线 |
| 采集资源 | 自然/医药 | 收集草药或制作材料 |

**探索流程**：
```
1. 玩家选择探索动作
2. 执行对应技能检定
3. LLM生成检定结果描写
4. 应用检定效果
5. 显示发现内容
6. 更新场景状态
```

**实现代码示例**：
```python
async def handle_explore(event: AstrMessageEvent, action: str):
    """处理探索动作"""
    skill_map = {
        "搜索": "investigation",
        "观察": "perception",
        "采集": "nature"
    }

    skill = skill_map.get(action)
    if not skill:
        yield event.plain_result("⚠️ 无效的探索动作")
        return

    # 执行技能检定
    result = skill_checker.roll_skill_check(player, skill)

    # 生成LLM描写
    scene_desc = await llm_generator.describe_explore(
        action=action,
        check_result=result,
        scene_context=current_scene
    )

    # 应用效果
    if result.success:
        yield event.plain_result(scene_desc + "\n" + discovered_content)
    else:
        yield event.plain_result(scene_desc + "\n" + nothing_found)
```

#### 10.1.3 解谜环节 🧩

**定义**：通过智力检定解开谜题机关，获取前进道路或宝藏。

**谜题类型**：

| 类型 | 描述 | 检定 | 难度 |
|------|------|------|------|
| 符文排序 | 按正确顺序激活符文 | 奥秘 | DC 15 |
| 机关破解 | 找到机关并正确操作 | 调查 | DC 12 |
| 密码破译 | 破解文字密码 | 奥秘 | DC 18 |
| 谜语问答 | 回答NPC的谜语 | 智力 | DC 14 |
| 陷阱排除 | 安全拆除陷阱 | 巧手 | DC 16 |

**解谜流程**：
```
1. 展示谜题描述
2. 玩家选择尝试解谜或寻找线索
3a. 直接解谜 → 智力检定
3b. 寻找线索 → 调查检定，成功则降低难度
4. 检定成功 → 解锁前进/获得奖励
5. 检定失败 → 触发负面效果（伤害/陷阱/怪物）
```

**提示系统**：
- 初始提供1个模糊提示
- 再次失败后提供更明确提示
- 提示次数影响最终奖励折扣

#### 10.1.4 战斗环节 ⚔️

**战斗流程**：

**战斗动作**：

| 动作 | 指令 | 效果 |

| 防御 | `/dnd 防御` | 本回合+2AC，下回合获得反应 |
| 施法 | `/dnd 施法` | 释放法术 |
| 移动 | `/dnd 移动` | 改变位置或脱离战斗 |
| 借机 | `/dnd 借机` | 敌人移动时触发借机攻击 |
| 躲避 | `/dnd 躲避` | 目标AC-2，敌人攻击劣势 |
| 准备 | `/dnd 准备` | 准备动作，指定触发条件 |
| 协助 | `/dnd 协助` | 帮助队友，下回合+2检定 |

**战斗规则**：
- 回合时间限制：60秒无操作则跳过
- 逃跑需要敏捷检定 DC=敌人AC
- 敌人全灭或队伍HP归零则战斗结束

#### 10.1.5 剧情环节 📖

**定义**：通过LLM生成NPC对话和剧情选择，影响副本走向。

**剧情触发点**：
- 进入特定场景
- 遇见特定NPC
- 完成关键目标
- 做出重要选择

**对话系统**：
```
玩家: /dnd 对话 [内容]
    ↓
意图识别 → 提取关键词
    ↓
规则校验 → 判断是否需要检定
    ↓
LLM生成 → 生成NPC回应
    ↓
效果应用 → 更新状态/触发事件
```

**剧情选择**：
- 每个关键剧情点提供2-4个选项
- 选择影响后续剧情分支
- 选择可能影响角色阵营
- 某些选择有道德/利益权衡

#### 10.1.6 奖励环节 🎁

**定义**：副本完成后结算奖励，包括经验、金币、装备等。

**奖励类型**：

| 类型 | 计算方式 | 说明 |
|------|----------|------|
| 经验值 | 敌人等级×50 + Boss×100 | 用于角色升级 |
| 金币 | 基础100 + 难度加成 | 可在商店使用 |
| 装备 | 随机掉落+等级匹配 | 提升角色战力 |
| 道具 | 特殊场景获取 | 消耗品或任务道具 |
| 声望 | 副本完成+难度 | 影响NPC态度 |

**奖励计算示例**：
```python
def calculate_rewards(dungeon: Dungeon, party: List[Character]) -> Rewards:
    """计算副本奖励"""
    base_xp = sum(enemy.level * 50 for enemy in dungeon.enemies)
    base_gold = 100 + dungeon.difficulty * 20

    party_bonus = len(party) * 10  # 组队加成

    equipment_drop = EquipmentLoot.roll(
        dungeon.difficulty,
        party[0].level,
        drop_rate=0.3
    )

    return Rewards(
        experience=base_xp + party_bonus,
        gold=base_gold,
        equipment=equipment_drop
    )
```


```python
try:
    result = some_function()

    yield event.plain_result("⚠️ 发生未知错误，请联系管理员")
```
