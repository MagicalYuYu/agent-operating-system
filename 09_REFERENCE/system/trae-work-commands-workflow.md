## TL;DR

- 命令是快捷方式，用于快速执行重复性任务
- 内置命令：`/plan`（计划模式）、`/spec`（规范模式）、`/browser_use`（浏览器操作）
- 自定义命令名称仅支持小写字母/数字/短横线
- Spec 工作流生成三阶段文档（spec.md/tasks.md/checklist.md），存储在 `.trae/specs/` 目录
- Plan 工作流生成 plan.md，存储在 `.trae/documents/` 目录
- Spec 适用于复杂系统级任务，Plan 适用于中小型功能

## 核心概念

### 命令系统

将常用 Prompt 封装为快捷指令，支持三种使用场景：

1. **复用常用 Prompt**：将经常使用的提示词封装为命令，避免重复输入（如总结 PR 变更、生成文档摘要）
2. **规范输出格式**：通过命令固定 AI 的输出结构或模板，确保生成内容符合团队规范（如 Conventional Commits）
3. **自动化常见开发流程**：将多步骤或复杂的指令封装为一个命令，一次触发即可执行（如代码审查、性能分析）

命令的运行环境：

| 环境类型 | 适用任务 | 适用客户端 |
|---|---|---|
| 本地 | 仅对本地任务生效 | TRAE Work 桌面版 |
| 云端 | 仅对云端任务（及从 GitHub 拉取的项目）生效 | TRAE Work 网页版、桌面版 |

### Spec 工作流

面向更复杂的系统级任务。AI 生成完整的三阶段文档组：
- **大纲**（spec.md）
- **任务列表**（tasks.md）
- **验收清单**（checklist.md）

文档存储在项目根目录下的 `.trae/specs/` 目录中，按任务名称分组为独立文件夹。首次创建时 AI 暂停等待确认，可编辑后确认执行。确认执行后，任务列表和验收清单的状态会随着执行进度自动更新。

### Plan 工作流

适用于中小型功能开发和模块级重构。AI 生成一份计划文档（plan.md），存储在 `.trae/documents/` 目录下，待确认后一一执行。若计划不符合预期，可直接手动编辑或向 AI 发送修改需求。

## 方法论 / 流程

### 创建自定义命令

1. 登录 TRAE Work
2. 进入技能与命令管理面板（两种方式）：
   - 点击 **头像** > **设置** → 左侧导航栏选择 **命令**
   - 在对话框内左下方点击命令图标 → 点击 **管理技能与命令**
3. (仅 TRAE Work 桌面版) 选择命令的运行环境：**本地** / **云端**
4. 在 **命令** 面板中，点击 **创建** 按钮
5. 配置命令参数，然后点击 **确认**

命令配置参数：

| 字段 | 描述 | 示例 |
|---|---|---|
| 命令名称 / Command Name | 命令的唯一标识，用于在对话中触发该命令。仅支持小写字母、数字和短横线（-）。 | summarize-pr-info |
| 描述 / Description | 对命令用途的简要说明。 | 总结 PR 信息。 |
| 说明 / Instructions | 定义触发命令时 AI 应执行的具体操作。建议清晰描述执行步骤、上下文来源以及输出内容。 | 查看当前 Pull Request 的代码变更内容，对比修改前后的代码，并总结本次 PR 的主要变更。输出内容包括：本次 PR 的核心改动点。主要修改的文件或模块。关键逻辑变化或新增功能。可能影响的功能或潜在风险。 |

### 使用命令

1. 在对话框中，输入 `/` 或点击左下方的命令图标，然后从菜单中选择一个命令
2. 输入与该命令作用相关的指令，然后发起对话

### 使用 Spec 工作流

启用方式：
- 在对话输入框中输入 **/**，然后从列表中选择 **Spec**
- 在对话输入框中输入 `/spec` 命令，然后按下回车键

Spec 工作流的使用场景：

| 场景 | 描述 | 为什么需要 Spec 工作流 |
|---|---|---|
| 从零搭建新系统/模块 | 从零搭建一个完整的服务、模块或应用 | 需求范围大、架构决策多，必须先对齐大纲再着手开发 |
| 大规模重构 | 对现有系统进行架构级重构或技术栈迁移 | 涉及大量文件和模块间依赖，需要清晰的任务拆解和验收标准 |
| 多人协作项目 | 团队中多位工程师基于 AI 协同开发同一项目 | Spec 文档作为单一事实来源对齐所有人的理解 |
| 高质量/高稳定性项目的开发 | 核心业务逻辑、支付系统、安全模块等关键功能的开发 | 需要详尽的验收清单来确保每个环节达标 |
| 需长期维护的项目 | 需要持续迭代、有历史包袱的成熟项目 | Spec 文档留存为项目知识资产，降低后续维护成本 |

### 使用 Plan 工作流

启用方式：
- 在对话输入框中输入 **/**，然后从列表中选择 **Plan**
- 在对话输入框中输入 `/plan` 命令，然后按下回车键

## 可复用经验

- Spec 的三阶段文档（大纲/任务/验收）适用于任何复杂项目，可作为项目知识资产长期保留
- Plan 的轻量级计划适用于快速迭代
- 命令系统可将团队 SOP 封装为可复用指令
- Spec 文档首次创建时 AI 暂停等待确认，可在此阶段充分编辑

## 关键坑点

- 命令名称仅支持小写字母/数字/短横线
- Spec 文档首次创建时 AI 暂停等待确认，确认后自动执行
- Plan 文档可手动编辑或让 AI 修改
- Spec 工作流生成的文档存于 `.trae/specs/`，Plan 工作流生成的文档存于 `.trae/documents/`

## 可迁移到 AOS 的规则

- Spec 三阶段文档模式可直接用于 AOS 的 Planner/Maker/Checker 流程
- 命令系统可复用为 AOS 的快捷指令机制
- `/spec` 和 `/plan` 的触发模式适用于 Agent 任务管理
- Spec 的验收清单（checklist.md）可作为 AOS 任务完成的自动化验证标准
