# AOS v1.1.0 Release Notes

> 发布日期：2026-06-26
> 版本：1.1.0

---

## 版本亮点

v1.1.0 是 AOS 的第二个正式版本，本次更新的核心是 **AOS Viewer 可视化界面**——让你的 AI 操作系统有了自己的"控制台"。

---

## 新增功能

### 🖥 AOS Viewer 可视化界面

AOS 自带一个 Liquid Glass 玻璃拟态风格的仪表盘，让你一眼看清系统全貌：

- **只读仪表盘**：系统总览 / Skill 管理 / 项目工作区 / 记忆中心 / 知识库 / Loop 监控 / 日志查看器
- **控制台**：一键触发自检 / 迁移采集等内置脚本
- **三种启动方式**：
  - 打包 EXE（双击即可运行，无需安装 Python）
  - 桌面窗口模式（PyWebView）
  - 浏览器模式（直接打开 index.html）
- **i18n 国际化**：内置 zh-CN / en-US 双语切换
- **Spotlight 全局搜索**：搜索模块 / Skill / 项目 / 记忆 / 知识，含关键词高亮
- **菜单栏 + 下拉菜单**：文件 / 视图 / 工具 / 帮助 四大菜单
- **三种主题**：深色 / 浅色 / 极简
- **启动动画**：Boot Screen + 动态背景光晕

### 📊 数据生成脚本

新增 `aos_generate_data.py`，扫描 AOS 文件系统自动生成 data.js：

- 解析 `00_BOOT/SYSTEM_STATE.md` → 组件状态、运行时统计、事件日志
- 解析 `00_BOOT/SKILL_REGISTRY.md` → Skill 列表
- 扫描 `01_PROJECTS/` → 项目列表（AGENTS.md + STATUS.md + README.md）
- 扫描 `04_MEMORY/` → 记忆索引和内容
- 扫描 `09_REFERENCE/` → 知识库索引和内容

### 📋 巡检规范

新增系统级 + 项目级双层巡检规范：

- `00_BOOT/SYSTEM_INSPECTION.md` — 系统级巡检（独立专用会话执行）
- `00_BOOT/PROJECT_INSPECTION.md` — 项目级巡检（项目工作会话前置）
- 四类文件分类标准（A 冗余 / B 过时 / C 留存 / D 迁移）
- 统一的路径层级规范（项目名 → 模块名 → 类型 → 日期）

## 改进

### 项目类型识别

- 新增"项目集"识别：AGENTS.md 含"子项目清单" section 即自动判定为项目集
- 支持插件集、微服务集等多种项目集形态

### 项目状态解析

- 重写状态解析逻辑，支持 4 级优先级（元信息状态 → 当前阶段 → 阶段统计 → 全文搜索）
- 支持中文状态标记（✅完成 / 🔄进行中 / ⏳待办）和无 emoji 格式
- 支持英文状态标记（completed / pending / in_progress）

### 项目描述解析

- 跳过标题 / 引用 / 分割线 / 徽章 / 图片 / 目录 / 表格 / 代码块行
- 取第一个完整段落，截断到 300 字符

### 示例项目

新增 3 个精心制作的示例项目，演示不同项目类型的标准结构：

- `_example_cli_tool` — 单一项目（Python 日志分析工具，零外部依赖）
- `_example_game_localization` — 单一项目（游戏本地化，翻译文件 + 完整流程文档）
- `_example_plugin_suite` — 项目集（聊天机器人插件集合，含 3 个独立插件 + EventBus 通信）

### 文档体系

- 所有目录 README 统一更新为 v1.1.0
- 新增 5 个目录 README：01_PROJECTS / 05_CACHE / 06_LOGS / 08_INBOX / 99_ARCHIVE
- AOS Viewer 使用说明完善，补充打包 EXE 启动方式
- 跨平台适配说明（TRAE / Claude Code / Codex）

## 升级指南

### 从 v1.0.0 升级

v1.1.0 引入了 AOS Viewer 可视化界面、项目类型识别、巡检规范拆分等多项变更。为不同使用习惯的用户提供两种升级方式：

#### 方式一：Agent 辅助更新（推荐）

将 [AOS_UPDATE_PROMPT_1.1.0.md](../07_EXPORTS/aos_viewer/AOS_UPDATE_PROMPT_1.1.0.md) 的内容粘贴到 TRAE / Claude Code / Codex 对话框，Agent 将自动执行完整的 8 步升级流程：

1. 环境扫描（检测当前版本 / 用户项目 / 自定义规则）
2. 备份（含 MD5 清单，可回滚）
3. 获取 1.1.0 文件（支持 GitHub Release 下载 / git clone / 本地目录三种方式）
4. 合并修改文件（分级处理：L1 直接覆盖 24 新增 / L2 智能合并 12 修改 / L3 删除迁移 1 文件）
5. 版本号同步（14 处文件位置）
6. 生成 data.js（AOS Viewer 数据源）
7. 一致性自检
8. 验证 AOS Viewer

#### 方式二：手动升级

参照 [MIGRATION_GUIDE_1.1.0.md](../07_EXPORTS/aos_viewer/MIGRATION_GUIDE_1.1.0.md) 手动完成 10 步升级，含完整 PowerShell 命令示例、回滚方案和常见问题排查。

### 关键变更点

| 类别 | 处理方式 | 说明 |
|------|----------|------|
| `03_TOOLS/aos_viewer/` | 直接复制 | 1.0.0 不存在，整个目录新增 |
| `03_TOOLS/scripts/aos_generate_data.py` | 直接复制 | 数据生成脚本，1.0.0 不存在 |
| `AGENTS.md` | 智能合并 | 新增"项目内标准结构"章节，**保留你的自定义规则** |
| `00_BOOT/SYSTEM_STATE.md` | 智能合并 | 更新版本号至 1.1.0，保留你的运行时统计与事件日志 |
| `00_BOOT/DAILY_INSPECTION.md` | 删除并迁移 | 拆分为 `SYSTEM_INSPECTION.md` + `PROJECT_INSPECTION.md` |
| 14 处版本号引用 | 同步更新 | 1.0.0 → 1.1.0（详见升级指南中的版本号同步表） |

### 验证

升级完成后运行自检脚本：

```bash
python 03_TOOLS/scripts/aos_check.py
```

输出 `一致性验证：通过` 即表示升级成功。然后双击根目录 `AOS Viewer.exe` 启动可视化界面确认。
