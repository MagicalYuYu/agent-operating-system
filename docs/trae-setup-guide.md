# AOS Trae 配置指南

本指南面向新用户，详细说明如何在 Trae IDE 中配置 AOS（Agent Operating System）项目。按照以下步骤逐一操作即可完成全部配置。

---

## 1. 前置条件

在开始之前，请确认以下软件已安装：

| 软件 | 最低版本 | 用途 | 检查方式 |
|------|----------|------|----------|
| Trae IDE | 最新版 | 代码编辑与 AI 交互 | 访问 https://trae.cn 下载安装 |
| Git | 2.0+ | 克隆项目仓库 | 终端执行 `git --version` |
| Python | 3.8+ | 运行 AOS 自检脚本 | 终端执行 `python --version` |

如果某项未安装，请先完成安装再继续。

---

## 2. 获取与打开项目

### 2.1 获取 AOS

两种方式任选其一：

**方式一：Git 克隆**（推荐，方便后续更新）

打开终端（Windows 可使用 PowerShell），执行以下命令：

```bash
git clone https://github.com/MagicalYuYu/agent-operating-system.git
```

**方式二：下载压缩包**

在 GitHub 仓库页面点击 **Code → Download ZIP**，解压到任意位置即可。AOS 是纯文件结构，无需安装任何依赖。

获取完成后，记住项目所在的本地路径，例如 `D:\agent-operating-system`。

### 2.2 在 Trae 中打开项目

1. 启动 Trae IDE
2. 点击左上角 **文件** 菜单
3. 选择 **打开文件夹**
4. 导航到刚才克隆的项目目录（即包含 `AGENTS.md` 的根目录），点击 **选择文件夹**
5. 确认 Trae 左侧文件树中能看到 `00_BOOT`、`01_PROJECTS` 等目录，说明打开正确

### 2.3 切换到 CODE 模式

1. 在 Trae 窗口右上角，找到模式切换按钮
2. 点击切换为 **CODE** 模式（而非 BUILD 模式）
3. CODE 模式下，AI 可以读取和修改项目文件，这是 AOS 正常运行的前提

## 3. 配置 Trae Rules（3 条规则）

Trae Rules 是项目级的 AI 行为约束规则。AOS 依赖 3 条规则来确保文件操作的一致性和规范性。以下逐条说明配置方法。

### 通用操作步骤

所有规则的添加入口相同：

1. 点击 Trae 左下角的 **齿轮图标**（设置），或使用快捷键打开设置
2. 在左侧导航栏中找到 **规则与记忆**
3. 点击 **规则** 标签页
4. 点击 **本地** 区域
5. 点击 **"+"** 按钮添加新规则

接下来为每条规则填写具体内容。

---

### 规则 1：aos-skill-rules

此规则约束 Skill 相关文件的操作规范，仅在 `03_TOOLS/skills/` 目录下生效。

| 字段 | 填写内容 |
|------|----------|
| 规则名称 | `aos-skill-rules` |
| 适用范围（路径限定） | `03_TOOLS/skills/` |

规则内容（完整复制以下文本）：

```
1. Skill 必须是文件夹结构，包含 SKILL.md（必须）+ gotchas.md（必须）+ 可选的 scripts/templates/references
2. SKILL.md 的 description 必须写触发条件而非摘要，不超过250字符
3. 每个 Skill 必须有 gotchas.md 坑点清单，持续积累
4. 新建 Skill 必须立即在 00_BOOT/SKILL_REGISTRY.md 注册
5. Skill 调用后必须更新 SKILL_REGISTRY.md 的 call_count
6. Skill 步骤序列的最后一步必须是状态更新（更新 STATUS.md / agent_tasks.json / SKILL_REGISTRY.md），不可省略
```

填写完成后，点击 **保存** 或 **确认**。

---

### 规则 2：aos-memory-rules

此规则约束记忆文件的操作规范，仅在 `04_MEMORY/` 目录下生效。

| 字段 | 填写内容 |
|------|----------|
| 规则名称 | `aos-memory-rules` |
| 适用范围（路径限定） | `04_MEMORY/` |

规则内容（完整复制以下文本）：

```
1. 记忆与指令必须分开存储
2. Memory 区分记录型与状态型：记录型（feedback/经验/坑点/操作日志）只追加不覆盖；状态型（用户画像/项目事实/系统配置）覆盖旧值，保留变更注释
3. INDEX.md 是索引不是内容，硬上限200行/25KB
4. feedback 每条必须包含三要素：规则+Why+How to apply
5. feedback 必须同时记录成功和失败
6. reference 只存指针不存内容
7. project 类型相对日期必须转绝对日期
8. 可推导信息不存入记忆
```

填写完成后，点击 **保存** 或 **确认**。

---

### 规则 3：aos-consistency-guard

此规则约束文件修改时的一致性检查，全局生效（不限定路径）。

| 字段 | 填写内容 |
|------|----------|
| 规则名称 | `aos-consistency-guard` |
| 适用范围（路径限定） | 留空（不填写，表示全局生效） |

规则内容（完整复制以下文本）：

```
1. 版本号同步：如果修改了任何文件中的版本号，必须扫描全部 .md 和 .json 文件中的版本引用，逐一更新至新版本号。归档文件 99_ARCHIVE/ 除外。
2. 交叉引用同步：如果新增、删除或重命名了文件，必须更新所有引用该路径的文件。
3. 状态同步：如果系统状态发生变化，必须同步更新 00_BOOT/SYSTEM_STATE.md 和 04_MEMORY/project/ 下的对应文件和 04_MEMORY/INDEX.md。
4. 自检验证：每次结构性变更完成后，必须扫描验证一致性，输出验证结果。
```

填写完成后，点击 **保存** 或 **确认**。

---

### 验证规则配置

3 条规则全部添加后，在 **规则与记忆 → 规则 → 本地** 页面应能看到以下条目：

| 规则名称 | 适用范围 |
|----------|----------|
| aos-skill-rules | 03_TOOLS/skills/ |
| aos-memory-rules | 04_MEMORY/ |
| aos-consistency-guard | （全局） |

如果缺少任何一条，请按上述步骤补充。

## 4. 配置自定义命令（3 条命令）

自定义命令是 AOS 的快捷操作入口，输入命令名称即可触发预设的 AI 行为。

### 通用操作步骤

所有命令的添加入口相同：

1. 点击 Trae 左下角的 **齿轮图标**（设置），或使用快捷键打开设置
2. 在左侧导航栏中找到 **规则与记忆**
3. 点击 **命令** 标签页
4. 点击 **"+"** 按钮添加新命令

接下来为每条命令填写具体内容。

---

### 命令 1：/aos-status

此命令用于快速查看 AOS 系统当前状态。

| 字段 | 填写内容 |
|------|----------|
| 命令名称 | `aos-status` |

命令内容（完整复制以下文本）：

```
读取 00_BOOT/SYSTEM_STATE.md 和 04_MEMORY/INDEX.md，输出 AOS 当前系统状态摘要，包括：系统状态、已注册 Skill 数量、记忆索引条数、待办任务数。
```

填写完成后，点击 **保存** 或 **确认**。

---

### 命令 2：/aos-ingest

此命令用于将网页知识入库到 AOS。

| 字段 | 填写内容 |
|------|----------|
| 命令名称 | `aos-ingest` |

命令内容（完整复制以下文本）：

```
执行 Web Knowledge Ingestion Skill (SK_0001)。用户提供 URL，按 Skill 流程完成：网页抓取 → 信息压缩与重组 → 入库（写入 08_INBOX/raw/ 和 09_REFERENCE/web/）→ 更新索引。
```

填写完成后，点击 **保存** 或 **确认**。

---

### 命令 3：/aos-review

此命令用于对当前会话的工作执行审查。

| 字段 | 填写内容 |
|------|----------|
| 命令名称 | `aos-review` |

命令内容（完整复制以下文本）：

```
对当前会话中完成的工作执行 VERIFY 模式审查：检查文件修改是否符合 AOS 规范、版本号是否一致、索引是否更新、状态是否同步。输出审查报告。
```

填写完成后，点击 **保存** 或 **确认**。

---

### 验证命令配置

3 条命令全部添加后，在 **规则与记忆 → 命令** 页面应能看到以下条目：

| 命令名称 | 用途 |
|----------|------|
| aos-status | 查看系统状态 |
| aos-ingest | 网页知识入库 |
| aos-review | 会话工作审查 |

如果缺少任何一条，请按上述步骤补充。

## 5. 验证安装

配置完成后，运行 AOS 自检脚本验证整体一致性。

### 5.1 运行自检

在 Trae 的终端中执行：

```bash
python 03_TOOLS/scripts/aos_check.py
```

### 5.2 预期输出

脚本会依次检查以下四项：

1. 版本号一致性
2. 引用完整性
3. 索引对应性
4. 目录合规性

如果全部通过，输出末尾会显示：

```
一致性验证：通过
```

### 5.3 如果验证失败

脚本会列出所有不一致的条目。常见原因及处理方式：

| 问题类型 | 可能原因 | 处理方式 |
|----------|----------|----------|
| 版本号不一致 | 文件中残留旧版本引用 | 执行 `python 03_TOOLS/scripts/aos_check.py --fix` 尝试自动修复 |
| 引用文件不存在 | 文件被移动或删除但引用未更新 | 手动检查并修正引用路径 |
| 索引缺失 | 新增文件未在索引中注册 | 在对应的 `_index.md` 或 `INDEX.md` 中补充条目 |
| 目录不合规 | 项目目录缺少必要文件 | 按提示补充 `README.md` 等必要文件 |

## 6. 常见问题

### Q1：Trae 没有自动加载 AGENTS.md 怎么办

AGENTS.md 是 AOS 的核心配置文件，位于项目根目录。Trae 在 CODE 模式下会自动读取工作区根目录的 AGENTS.md 作为项目级指令。

排查步骤：

1. 确认 `AGENTS.md` 文件位于项目根目录（与 `00_BOOT` 同级）
2. 确认当前处于 **CODE 模式**，而非 BUILD 模式
3. 尝试关闭项目后重新打开：**文件 → 关闭文件夹**，然后重新 **打开文件夹**
4. 如果仍然不生效，在对话中直接告诉 AI："请读取 AGENTS.md 并遵循其中的规则"，手动触发加载

### Q2：规则不生效怎么排查

规则不生效时，按以下顺序排查：

1. **检查规则是否存在**：进入 **设置 → 规则与记忆 → 规则 → 本地**，确认 3 条规则均已添加
2. **检查路径限定是否正确**：
   - `aos-skill-rules` 的路径限定必须是 `03_TOOLS/skills/`
   - `aos-memory-rules` 的路径限定必须是 `04_MEMORY/`
   - `aos-consistency-guard` 的路径限定必须留空
3. **检查规则内容是否完整**：逐条对比本指南中的规则内容，确认没有遗漏或截断
4. **重启对话**：规则在新建对话时加载，当前对话中添加的规则可能需要开启新对话才能生效
5. **手动触发**：在对话中明确告诉 AI："请遵循 aos-skill-rules 规则"，验证 AI 是否理解规则内容

### Q3：自检脚本报错怎么处理

| 报错信息 | 原因 | 解决方法 |
|----------|------|----------|
| `No module named 'xxx'` | Python 缺少依赖 | 自检脚本仅使用标准库，不应出现此错误。检查 Python 版本是否为 3.8+ |
| `FileNotFoundError` | 脚本执行路径不对 | 确保在项目根目录下执行命令，或使用完整路径 `python D:\agent-operating-system\03_TOOLS\scripts\aos_check.py` |
| `PermissionError` | 文件权限不足 | 检查项目目录的读写权限，确保当前用户有访问权限 |
| `UnicodeDecodeError` | 文件编码问题 | 确保项目中的 .md 文件使用 UTF-8 编码 |

如果遇到以上未覆盖的问题，请记录完整的错误信息，在项目 GitHub 仓库提交 Issue。
