# AOS v1.1.0 — 99_ARCHIVE

不可修改的历史归档区，归档内容只读。

## 应放什么

- 已归档的项目历史版本
- 超过保留期的备份文件（从项目 `backup/` 迁移而来）
- 已废弃但仍需保留追溯性的文件
- 被新版本替代的旧版设计文档
- 早期开发过程中的快照备份

## 禁止放什么

- 当前活跃使用的文件
- 运行时状态（应放 04_MEMORY/）
- 待处理的外部输入（应放 08_INBOX/）

## 分层规则

按"项目名 → 类别 → 日期或版本"分层存放：

```
99_ARCHIVE/
├── aos_v1.0.0-dev2-original-design.md   ← 早期设计文档归档
├── {project_name}/                      ← 按项目名分层
│   ├── backup_zips_202605/              ← 按类别 + 日期分层
│   │   └── *.zip
│   ├── dnd_zero_dev_20260509/           ← 开发备份归档
│   │   └── *.py / *.md
│   └── archived_plugins/                ← 归档插件
│       └── {plugin_name}_v{version}/
└── README.md
```

## 注意

- 归档内容**只读**，禁止修改
- 归档操作必须有明确记录（写入对应项目的 STATUS.md 和项目记忆）
- 删除操作前必须先归档至本目录
- 归档文件不参与一致性检查（版本号扫描排除 99_ARCHIVE/）
