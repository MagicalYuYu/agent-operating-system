## TL;DR

- GitHub 集成：仅 Code 模式，支持 AI 创建/检查 PR
- 飞书集成：授权连接，无需配置 MCP，支持云文档/多维表格/日历/消息等
- 云端运行环境：自定义预装依赖 + 环境变量 + 敏感变量加密 + 运行时配置 + 网络策略
- 工作树：每个任务独立 Git 目录，仅本地环境
- 自动化：定时任务（固定/间隔/自定义），三种创建方式
- 语音讨论：多轮语音交互，自动转写清洗结构化
- 沙箱：受限执行环境，高风险命令拦截，多模式权限控制
- 对话流设置：自动运行方式 + 任务状态通知
- 隐私模式：开启后对话内容不用于训练
- 云端智能体数据存储：代码库数据不用于模型训练，删除会话后永久删除数据

## 一、GitHub 集成

### 前置条件

仅 Code 模式支持 GitHub 应用。

### 集成步骤

1. 在界面左下角，点击 **头像** > **设置**，进入设置中心
2. 在左侧导航栏中，选择 **外部应用授权** → 进入外部应用集成面板
3. 在 GitHub 应用处，点击 **连接** 按钮
4. 根据页面提示，登录 GitHub 账号，然后授权 TRAE-AI CN 访问账号信息和代码仓库
5. 授权完成后，GitHub 应用的集成方中会显示 Web 和 Desktop

### 创建 PR

**准备工作：选择分支并发起对话**

- 网页版：在对话框左下方的 **选择仓库** 处选择需要创建 PR 的仓库和分支
- 桌面版：打开一个从 GitHub 拉取的项目，在对话框左下方选择 **云端** 模式，然后选取需要创建 PR 的分支

**由 AI 创建 PR：**

1. 点击 **AI 创建 PR** 按钮
2. AI 查看两个分支间的具体差异，生成摘要报告
3. 可点击摘要右侧的箭头按钮，在 DiffView 中查看每个文件具体的代码变更
4. 确认代码变更无误后，在 **接下来您想做什么？** 对话框中，选择 **创建 PR**
5. AI 将为选中的仓库和分支创建 PR，并打开 GitHub 窗口

**手动创建 PR：**

1. 点击 **···** > **手动创建 PR**
2. 系统自动打开 GitHub 窗口，展示 PR 信息

### AI 检查 PR

点击对话框上方的 **AI 检查 PR** 按钮，AI 会先查看相关文件的具体变更内容，然后执行 `git diff` 命令来获取详细的变更信息。

## 二、飞书集成

### 使用场景

- **需求文档驱动的代码开发**：直接引用飞书中的 PRD、技术方案或接口文档，由 AI 解析文档内容并转化为上下文
- **多维表格驱动的任务与数据管理**：将多维表格作为任务看板或数据源，AI 自动读取任务记录、更新执行状态
- **团队协作与信息同步自动化**：创建日程安排评审或联调会议，将会议纪要同步至云文档

### 集成步骤

1. 在界面左下角，点击 **头像** > **设置**，进入设置中心
2. 在左侧导航栏中，选择 **外部应用授权**
3. 在飞书应用处，点击 **连接** 按钮 → 前往飞书权限授权页
4. 选择需要授权给 TRAE Work 的飞书权限
5. 点击 **授权并连接飞书** 按钮 → 前往授权确认页面
6. 确认飞书账号及可授予的权限，阅读注意事项，点击底部的 **授权** 按钮
7. 授权完成后，TRAE Work 会自动安装本地 CLI 及相关技能

## 三、云端运行环境

### 配置参数

| 参数 | 描述 |
|---|---|
| 环境名称 / Environment Name | 环境的基本信息，用于标识和区分不同的开发环境 |
| 预装依赖 / Preinstalled Packages | 定义基础镜像和预装的语言版本，默认使用最新版本 |
| 环境变量 / Environment Variables | 程序运行时的配置，保存普通配置（如接口地址或运行模式） |
| 敏感变量 / Sensitive Variables | 存储敏感信息，所有敏感变量都会被加密存储，仅在运行时使用 |
| 运行方式 / Runtime Configuration | 指定启动命令（如 `npm run dev`） |

### 预装语言和版本

| 语言 | 字段 | 支持的版本 |
|---|---|---|
| Python | `python_version` | 3.10、3.11、3.12、3.13、3.14 |
| Node.js | `node_version` | 18、20、22、24 |
| Go | `go_version` | 1.22.12、1.23.8、1.24.3、1.25.1 |
| Rust | `rust_version` | 1.83.0 ~ 1.92.0 |
| Java | `java_version` | 11、17、21、22、23、24、25 |
| Ruby | `ruby_version` | 3.2.3、3.3.8、3.4.4 |
| PHP | `php_version` | 8.2、8.3、8.4、8.5 |
| Swift | `swift_version` | 5.10、6.1、6.2 |

注意：若未手动指定版本则默认最新；可通过 `GET /api/v1/environment_configs/supported_versions` 获取最新版本列表；暂不支持自定义容器镜像。

### 敏感变量 vs 环境变量

| | 敏感变量 | 环境变量 |
|---|---|---|
| 存储方式 | KMS 加密存储 | 明文 JSON 存储 |
| API 返回 | 仅返回 key 列表，不返回值 | 返回完整 key-value |
| 适用内容 | API 密钥、数据库密码、访问令牌 | DEBUG 开关、LOG_LEVEL、端口号 |
| 安全级别 | 高 | 低 |
| 数量限制 | 最多 50 个 | 最多 100 个 |

### 运行时配置

| 参数 | 作用 | 相关限制 | 执行方式 |
|---|---|---|---|
| 安装脚本 / Install Script | 环境启动时自动执行，通常用于安装依赖 | `install` 命令长度不得超过 10KB | 代码 clone 完成后，阻塞执行，等待完成 |
| 启动脚本 / Start Script | 依赖安装完成后执行，用来启动项目 | `start` 命令长度不得超过 10KB | 安装脚本执行完，后台执行，不阻塞 |
| 默认命令行 / Default Shell | 配置与主程序并行运行的后台任务 | 最多同时打开 10 个终端，`terminals` 命令长度不得超过 4KB | 并行打开所有终端 |

脚本执行顺序：
1. **install**（阻塞等待，必须完成后才继续）
2. **start**（后台运行，不阻塞）
3. **terminals**（并行打开所有终端）

注意：`install` 失败会阻断后续步骤；`start` 和 `terminals` 都是长时间运行的进程；所有 `terminals` 并行启动，无先后顺序。

### 网络策略配置

暂不支持自定义网络策略。通过 `network_policy` 对象限制出站流量：

| 参数 | 描述 |
|---|---|
| mode | 网络策略开关，固定设置为 `1`，表示启用 |
| allowlist_policy | 白名单策略模式，固定设置为 `0`，表示使用 `common_dependencies` 列表作为白名单 |
| common_dependencies | 字符串数组，指定允许访问的依赖源。可选值：`npm`、`pypi`、`maven`、`goproxy`、`rubygems`、`packagist`、`crates`、`docker`、`github`、`gitlab` |

### 完整配置示例

```json
{
    "name": "Full Stack Dev Environment",
    "description": "React + FastAPI 全栈开发环境",
    "preinstalled_packages": {
        "python_version": "3.12",
        "node_version": "20"
    },
    "runtime_config": {
        "install": "npm install && pip install -r requirements.txt",
        "start": "npm run dev",
        "terminals": [
            {
                "name": "Backend Server",
                "command": "uvicorn main:app --reload --port 8000",
                "description": "FastAPI 后端服务"
            },
            {
                "name": "Test Runner",
                "command": "pytest --watch",
                "description": "Python 测试监听"
            }
        ]
    },
    "secrets": {
        "API_KEY": "your_api_key",
        "DB_PASSWORD": "your_db_password"
    },
    "environment_variables": {
        "DEBUG": "true",
        "LOG_LEVEL": "info"
    },
    "network_policy": {
        "mode": 1,
        "allowlist_policy": 0,
        "common_dependencies": ["npm", "pypi", "github"]
    }
}
```

### 使用限制

自定义云端运行环境只能在 Code 模式中使用。

- **网页版**：直接在对话框的右下方选择所需的自定义云端运行环境
- **桌面版**：打开 GitHub 远程项目 → 对话输入框左下角设置 **云端** → 选择所需的自定义云端运行环境

## 四、工作树（Worktree）

### 名词解释

| 名词 | 描述 |
|---|---|
| 工作树 | Git 原生机制，允许单个代码仓库同时拥有多个独立的工作目录，每个目录可检出不同分支 |
| 工作树分支 | 某个工作树当前检出的 Git 分支（如 `feat-pet-shop-1sfbnt`） |
| 工作树目录 | 与工作树关联的本地文件夹路径，格式为 `/{worktree_branch_name}` |

### 适用范围

仅适用于本地环境中运行的任务。前置条件：本地已安装 Git。

### 创建工作树

发起任务前，在对话输入框左下角将模式设置为 **工作树**。任务启动后，智能体自动创建专属的工作树分支和目录。

### 合并代码

- **AI 自动合并**：点击 **AI 合并** 按钮，AI 自动分析代码变更、解决冲突并完成合并
- **手动合并**：点击 **···** > **手动合并**，按界面指引操作。若出现冲突，先点击 **AI 解决冲突** 按钮

代码合并后，对话流底部显示 "代码已合并" 分割线。分割线上方的对话内容无法回退。

### 清理工作树

清理工作树会删除本地目录释放磁盘空间，但对话历史和代码变更记录保留。可选择是否保留 Git 分支。

三种清理方式：
1. **从任务列表中清理**：悬停目标任务 → **···** → **删除工作树** → 勾选是否保留分支 → 删除
2. **从设置中心清理**：**设置** > **工作树** → 选择工作树 → **清理** → 勾选是否保留分支 → 删除
3. **从工作树分支栏中清理**：分支栏 **···** → **删除工作树** → 勾选是否保留分支 → 删除

### 相关设置

| 设置 | 描述 |
|---|---|
| 清理工作树时默认保留分支 | 启用后，删除工作树时确认弹窗默认勾选 **保留工作树分支** |
| 工作树磁盘空间占用提醒 | 设置磁盘空间占用阈值，达到时系统发出通知提醒清理 |

## 五、自动化

### 使用场景

- 代码仓库的定期维护（安全扫描、Bug 排查、测试用例补齐、变更日报）
- 信息检索与汇总（行业资讯、竞品动态、用户反馈归集）
- 周期性内容产出与汇报（舆情周报、运营日报、指标监控）

### 触发方式

- **固定时间**：按每天、每周、每月的固定时间点执行
- **间隔触发**：按固定时间间隔（分钟/小时/天）循环执行
- **自定义**：用自然语言描述定时策略（如 "工作日早上 9 点" 等），仅 "在对话中创建" 方式支持

### 创建自动化任务

1. 在左栏顶部，点击 **自动化** → 界面右侧显示自动化任务管理面板
2. 选择创建方式：

**在对话中创建：**

1. 点击右上角的 **在对话中创建** 按钮
2. 系统预填入模板：
   ```
   我想要创建一个自动化任务。
   任务内容是：
   执行时间是：
   ```
3. 输入任务内容和执行时间（运行模式默认为当前模式，且不支持修改）
4. 在输入框左下角，选择任务运行环境以及输出内容的存储位置
5. 点击 **发送** → AI 解读需求并创建任务
6. 检查 AI 输出的自动化任务信息，按需调整

**手动创建：**

1. 点击右上角的 **手动新建** 按钮
2. 配置任务：任务名称、触发时间、任务内容、运行模式（Work / Code）、运行环境（云端 / 本地）、输出内容的存储位置
3. 点击 **创建**

**从模板创建：**

1. 进入 **任务模板** 页签
2. 选择模板（弹窗中已预设完整配置）
3. 按需修改任务配置
4. 点击 **创建**

**重要限制**：任务一旦创建，无法再修改其运行模式、运行环境和输出内容的存储位置。

### 查看执行记录

在自动化任务管理面板的 **执行历史** 页签下查看。需确保 TRAE Work 当前的运行模式与任务的运行模式一致。点击任意记录可跳转到该任务的对话流页面。

## 六、语音讨论

### 使用场景

- 产品经理：从灵感到方案的头脑风暴（构思新产品、优化现有项目）
- 研发工程师：基于代码的技术讨论（Bug 分析、方案取舍）
- 产品运营：运营策略的多轮推演
- 数据分析师：围绕数据的对话式分析
- 通用场景：基于历史讨论的信息检索

### 操作步骤

1. 在对话输入框的右下角，点击 **语音讨论** 图标
2. 首次使用时，点击 **开始讨论** 并授权麦克风权限
3. (可选) 点击右上角 **开启字幕** 查看实时转录
4. 开始与 AI 讨论（表达完想法稍作停顿后 AI 给出反馈，可随时暂停/恢复语音输入）
5. 讨论完成后，点击 **结束** → AI 整理为结构化文本，展示在 **讨论结果** 面板
6. 可点击 **查看会议记录** 查看原始讨论记录

### 文本处理策略

- **去噪与口语清洗**：移除 "嗯"、"然后"、"就是"、"那个" 等语气词和口癖词，合并语义重复，修复断句问题，修正语法错误
- **语义理解与自动修正**：识别自我修正行为（如 "不是 A，是 B"），保留最终结论，删除被推翻内容
- **结构化表达**：自动生成标题、整理要点为列表、按总分结构组织段落

## 七、沙箱

### 支持的操作系统

macOS（基于 `sandbox-exec` 工具）

### 桌面版 Code 模式 — 云端任务

所有云端任务运行于独立的安全沙箱容器内。文件访问权限：

| 权限类型 | 目录类型 | 权限范围 |
|---|---|---|
| 读写 | 当前会话的工作目录 | 仅当前项目或当前会话内创建、同步的文件与目录 |
| 读写 | 沙箱容器内的临时与缓存目录 | 仅当前会话运行所需的临时文件及依赖缓存路径 |
| 不可访问 | 宿主机的文件系统 | 无法访问用户本地设备或平台宿主机上的任何目录 |
| 不可访问 | 其他会话的数据目录 | 无法访问其他会话的文件、缓存、磁盘及运行环境资源 |

### 桌面版 Code 模式 — 本地任务

当前安全策略不涉及网络访问。读写权限继承当前用户权限；读写与只读冲突时以只读为准。

| 权限类型 | 目录类型 | 目录列表 |
|---|---|---|
| 只读 | 项目目录中的受保护目录 | `.vscode` |
| 只读 | 根目录 | `/`（默认策略：所有未显式声明为可写的目录均为只读） |
| 读写 | 项目目录 | 除 `.trae`、`.vscode` 和 `.git` 之外的项目文件与目录 |
| 读写 | 临时目录 | macOS: `/tmp`、`/var/folders`、`TMPDIR`；Windows: `~/AppData/Local/Temp`、`~/AppData/LocalLow/Temp`；Linux: `/tmp`、`TMPDIR` |
| 读写 | 缓存目录 | macOS: `~/Library/Caches`、`~/.cache`、`XDG_CACHE_HOME`；Linux: `~/.cache`、`XDG_CACHE_HOME` |
| 读写 | 通用工具依赖目录 | macOS & Linux: `~/.local/lib`、`~/.local/bin`、`~/.local/share`；Windows: `LOCALAPPDATA` 内常见工具依赖路径 |
| 读写 | 常用语言的工具链及其依赖目录 | Go、Java、Python、Node.js、Rust 和 C++ 等 |

### 桌面版 Work 模式

- **完全隔离**：AI 生成的所有代码和命令都在沙箱内执行
- **开箱即用**：沙箱镜像预装了完整的 Skills 工具链
- **安全共享**：项目文件通过受控方式挂载到沙箱中
- 文件访问控制：仅能读写当前项目的目录

### 网页版 Code 和 Work 模式

所有云端任务运行于独立的安全沙箱容器内。文件访问权限与桌面版 Code 模式云端任务相同。

### 启用沙箱

1. 点击 **头像** > **设置**
2. 选择 **对话流**
3. 在 **自动运行** 部分，将 **命令运行方式** 设置为 **沙箱运行（支持白名单）**
4. (可选) 在 **白名单列表** 处将命令前缀加入白名单（白名单命令跳过沙箱直接执行）

### 高风险命令的运行策略

**普通命令：**
- 在白名单内 → 沙箱外运行
- 不在白名单内 → 沙箱内运行，失败后询问是否在沙箱外重试

**高风险命令（如 `rm -rf`）：**
系统拦截并弹出提示，可选：
- **跳过**：不运行
- **添加到白名单**：将命令前缀添加到白名单
- **运行**：本次在沙箱内运行

## 九、隐私模式

开启后，TRAE Work 不会将对话内容（包括代码片段及 AI 生成的输出结果）用于数据分析、产品优化或模型训练。

### 重要提示

- 隐私模式仅在登录 TRAE 账号时生效，退出登录后自动关闭
- 无论隐私模式是否开启，代码库文件绝不用于数据分析、产品优化或模型训练
- 本地服务：代码库文件保存在本地，临时上传计算嵌入向量后永久删除明文
- 云端服务：代码库文件在云端处理并留存

### 启用方式

前往 **设置** > **账号** → 打开 **隐私模式** / **Privacy Mode** 开关

## 十、问题排查

### 工作环境启动失败

**macOS 通用解决方案：**

1. 使用 command + Q 快捷键完全退出 TRAE Work
2. 打开活动监视器，在 CPU 页签中搜索 "SOLO" 关键字
3. 结束 "TRAE SOLO CN 的" 进程（如果存在）
4. 检查磁盘空间和内存（确保剩余 2G 磁盘空间和 1G 可用内存）
5. 重新打开 TRAE Work

**Windows 通用解决方案：**

检查系统版本。若 OS Build 低于 19044，先升级至 Windows 10 21H2 或更高版本。

### 特定错误码解决方案

| 错误码 | 适用操作系统 | 问题 | 解决方案 |
|---|---|---|---|
| 992501 | macOS | 当前选择的项目目录不存在 | 确保项目目录存在 |
| 992503 | macOS | 无当前项目目录的访问权限 | 检查用户权限，前往隐私与安全性设置检查是否已将目录授权给 TRAE Work |
| 992602.xxxx | macOS、Windows | 工作环境启动失败 | 参考通用解决方案，重启 TRAE Work |
| 992607 | macOS | 工作环境加载宿主机文件时发生错误 | 先尝试通用方案；若无法解决，检查工作区目录是否已被删除 |
| 992608 | macOS | 工作环境配置错误 | 选择 **帮助** > **在文件夹中打开日志** → 进入 `ModularData/ai-agent/vm/` → 删除 `vms` 文件夹 → 完成通用解决方案步骤 |
| 992614 | macOS | 可用内存不足以启动工作环境 | 先尝试通用方案；若无法解决，释放内存后重试 |

## 十二、云端智能体数据存储声明

### 代码库数据

云端存储的数据包括：
- 授权访问的代码仓库内容
- 系统运行所需的规则与状态数据（MCP、规则、记忆、自定义运行环境、Hooks 及 Context 等）

### 对话历史数据

**会话存续期间：** 系统在云端保存代码数据、对话内容、上下文信息、会话维度指标信息。

**用户主动删除会话后：** 系统永久删除与该会话相关的所有数据（代码数据、历史对话、上下文信息、指标信息），不再保留。

## 可复用经验

- 飞书集成无需配置 MCP，授权即用，降低使用门槛
- 工作树模式避免并行任务代码冲突
- 自动化模板可快速创建常用定时任务
- 语音讨论的文本处理策略（去噪/修正/结构化）可复用
- 沙箱白名单机制平衡安全与效率

## 关键坑点

- GitHub 集成仅 Code 模式支持
- 工作树仅本地环境适用
- 自动化任务创建后运行模式/环境/存储位置不可修改
- 沙箱仅 macOS 支持 sandbox-exec
- 语音讨论需授权麦克风权限
- 隐私模式仅登录时生效，退出登录自动关闭
- 云端服务时代码库文件会在云端留存
- 云端运行环境仅 Code 模式可用
- 暂不支持自定义容器镜像和网络策略

## 可迁移到 AOS 的规则

- GitHub/飞书等外部应用授权模式可复用
- 工作树的 Git 隔离机制适用于 Agent 并行任务
- 自动化的定时任务模式可复用为 AOS Loop Engine
- 沙箱的权限控制模型适用于 Agent 安全执行
- 语音讨论的文本处理策略可复用为 AOS 输入预处理
- 云端运行环境的运行时配置模式（install → start → terminals）可复用为 AOS 环境编排
- 敏感变量 vs 环境变量的分层存储策略可复用
- 网络策略白名单模式可复用为 AOS 出站流量控制
