# Legacy Migration Skill (SK_0002)

> AOS Skill — 存量内容迁移接入

---

## 决策规则

1. 迁移前必须确认 AOS 根目录位置
2. 所有内容必须经过 08_INBOX/migration/ 中转
3. 脱敏处理是入库前的强制步骤，不可跳过
4. 不确定分类的内容放入 02_SANDBOX/migration_pending/
5. 关联分析必须覆盖端口映射、服务依赖、共享配置、API调用链
6. 对话提取由用户在目标窗口中手动触发，Agent 不自动提取
7. 迁移完成后必须更新所有相关索引和状态文件
8. 原始文件在迁移过程中只读，不修改

---

## 坑点

| 坑点 | 规避方式 |
|------|----------|
| Trae对话无法API批量导出 | 用户需逐个窗口粘贴预设提示词 |
| 项目间隐含关联难以发现 | 扫描配置文件、端口引用、环境变量 |
| 敏感信息泄露 | 入库前强制脱敏检查 |
| 同一知识多处重复 | 去重合并，保留最新版本 |
| 分类标准不明确 | 不确定的放入02_SANDBOX/ |
| 跨平台格式不统一 | 统一转为Markdown格式 |
| AOS路径因用户不同 | 脚本自动检测AOS根目录（向上查找AGENTS.md） |
| 服务器未关停就复制数据 | 数据可能不一致 | 先优雅关停服务，确认进程结束后再复制 |
| 备份未验证就迁移 | 备份可能损坏 | 备份后必须验证文件数/大小/校验和 |
| 敏感信息遗漏 | 迁移后服务无法启动 | 强制提取端口/接口/路径/环境变量清单 |
| 无回滚方案 | 迁移失败无法恢复 | 迁移前必须制定回滚方案 |
