> 主题：TRAE Work 产品概述

---

## TL;DR

- TRAE Work 是 AI 原生工作台，提供网页版/桌面版/移动版三端形态
- 设有 Work（非开发）和 Code（开发）双模式
- 云端智能体（Cloud Agent）提供统一云端运行时与依赖管理
- 三端共享同一套账号体系和任务数据，任务信息实时全量同步
- 移动版定位为跨设备任务下发中心，桌面版/网页版负责任务执行与产物验收

---

## 三端架构详解

### TRAE Work 网页版

适用场景：临时需求、外出办公、快速验证

| 特性 | 说明 |
|------|------|
| 访问方式 | 有网络即可访问，随时随地启动任务 |
| 安装要求 | 无需下载和安装，开箱即用的云端环境 |
| 资源占用 | 任务管理简洁高效，轻量化交互不占用本地资源 |
| 运行环境 | 仅支持云端运行环境 |

### TRAE Work 桌面版

适用场景：长期开发、复杂项目

桌面版是一款专为 AI Agent 设计的独立应用，不依赖于 TRAE IDE 运行，兼顾本地与云端任务。

| 特性 | 说明 |
|------|------|
| 输入方式 | 支持文字、语音、附件、技能等多元输入 |
| 进度展示 | 实时展示任务当前进度并自动总结输出内容，流程透明可控 |
| 预览验收 | 支持在对话界面内直接预览成果并完成验收，无需切换工具 |
| 性能优化 | 深度性能优化确保 Coding Agent 运行更稳定、效果更突出 |
| 独立运行 | 独立运行带来更强性能和更丰富交互拓展，流畅度大幅提升 |
| 运行环境 | 支持"本地"和"云端"两种运行环境 |

### TRAE Work 移动版

适用场景：碎片化场景下快速下发和管理任务

移动版将 TRAE Work 能力延伸至手机，使其成为口袋里的 AI 智能体调度中心。

| 特性 | 说明 |
|------|------|
| 语音输入 | 默认支持"按住说话"功能，无需打字即可下达指令；同时支持文本、附件和快捷指令 |
| 跨设备管理 | 统一管理云端和多台个人电脑中的 TRAE Work，支持向不同设备并发派发任务并实时监控 |
| 离线切换 | 设备离线时，系统自动切换至云端执行任务，确保任务不中断 |
| 状态同步 | 任务状态在移动版、网页版和桌面版之间无缝同步 |
| 全流程优化 | 深度优化任务管理、对话交互与产物预览全流程，提供完善的设备与账户设置 |

### 三端联动机制

三端共享同一套账号体系和任务数据，任务信息实时在三端全量同步。

| 端 | 角色 | 职责 |
|----|------|------|
| 移动版 | 任务下发中心 | 在碎片化场景下快速下发和管理任务 |
| 网页版 | 任务执行与验收 | 负责任务执行、深度交互和产物验收 |
| 桌面版 | 任务执行与验收 | 负责任务执行、深度交互和产物验收 |

联动架构：
- 移动版 → 下发云端任务 → 云端沙箱 → 智能体自动执行
- 移动版 → 下发本地任务 → 桌面版（公司电脑/家庭电脑）
- 三端之间实时同步，推送结果

## 双模式：Work & Code

| 模式 | 目标用户 | 适用场景 |
|------|----------|----------|
| Work 模式 | 非开发用户（产品经理、数据分析师、运营等） | 处理文档、数据、演示稿等 |
| Code 模式 | 开发工程师（习惯于智能体主导开发） | 编码、调试、代码库管理和 Git 工作流 |

---

## 云端智能体（Cloud Agent）

云端智能体让用户在云端环境中执行代码分析、运行和调试等任务。

| 特性 | 说明 |
|------|------|
| 运行时管理 | 提供统一的运行时与依赖管理 |
| 环境隔离 | 所有代码在稳定且隔离的远程环境中完成 |
| 兼容性保障 | 有效避免本地环境差异带来的兼容性或性能问题 |
| 专注任务 | 无需为环境配置或设备限制分心 |

---

## 关键坑点

- 移动版仅支持预览云端任务生成的 Web 页面，本地任务 Web 页面需用桌面版/网页版查看
- iOS 版尚未在美区 App Store 上架，美区 Apple ID 无法下载
- 网页版仅支持云端运行环境，不支持本地运行环境

## 可迁移到 AOS 的规则

- 多端协同需要统一账号体系和数据同步机制
- 任务下发与执行分离的架构模式可复用
- 云端/本地双运行环境的设计思路适用于 Agent 系统
- 三端联动架构（移动端下发、桌面/网页端执行、云端提供运行时）可作为多端 Agent 系统的参考模型
