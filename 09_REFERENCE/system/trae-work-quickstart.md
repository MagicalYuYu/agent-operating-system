# TRAE Work 快速开始

> 来源：https://docs.trae.cn/solo_trae-solo-quickstart + https://docs.trae.cn/solo_get-started-with-trae-solo-mobile
> 入库时间：2026-06-21T22:00:00+08:00
> 主题：TRAE Work 快速开始（网页版/桌面版/移动版）
> 标签：trae, work, quickstart, web, desktop, mobile, 任务管理, 预览迭代

---

## TL;DR

- 网页版/桌面版支持 macOS 和 Windows
- 工作区四面板：对话面板、待办、任务产物、参考信息
- Web 应用支持浏览器预览 + 元素选择迭代
- 移动版定位为跨设备任务下发中心，支持语音/文本/附件/技能输入
- 移动版连接电脑后有效期 180 天，支持"保持电脑唤醒"
- 三端任务状态实时同步，App 后台/断网不影响任务执行
- 网页版仅支持云端运行环境，桌面版支持本地+云端双环境

## 第一部分：网页版和桌面版快速开始

### 获取与安装

| 形态 | 获取方式 |
|------|----------|
| 网页版 | 访问 https://work.trae.cn/ |
| 桌面版 | 前往 TRAE Work 桌面版下载页下载 |

支持的操作系统：macOS、Windows

### 模式选择

打开 TRAE Work 后，在界面左上角切换工作模式：

| 模式 | 目标用户 | 适用场景 |
|------|----------|----------|
| Work 模式 | 非开发用户（产品经理、数据分析师、运营等） | 处理文档、数据、演示稿等 |
| Code 模式 | 开发工程师（习惯于智能体主导开发） | 编码、调试、代码库管理和 Git 工作流 |

### 发起任务

在主界面的对话框中，选择一个项目，输入任务指令，然后发送给 AI。

**网页版特殊行为**：可直接发送指令，TRAE Work 网页版会自动创建项目，并根据指令中的关键词命名项目和任务。

| 维度 | 支持内容 |
|------|----------|
| 输入形式 | 文字、语音 |
| 上下文 | 附件（包括 .pptx、.xlsx 等）、图片、斜杠命令、技能 |
| 运行环境（网页版） | 仅支持云端运行环境 |
| 运行环境（桌面版） | 支持"本地"和"云端"运行环境 |

### 工作区四面板

发起任务后进入工作区，包含以下面板：

| 编号 | 面板名称 | 描述 |
|------|----------|------|
| 1 | 对话面板 | 与 AI 实时对话，下达指令并查看输出 |
| 2 | 待办 | 当任务较复杂时，AI 列出任务计划（Todos），可实时追踪每个子任务进度 |
| 3 | 任务产物 | 存放 AI 为该任务生成的文件 |
| 4 | 参考信息 | 实时显示 AI 执行任务时引用的所有上下文（技能、文件、网络信息等）；若上下文超出预期，可点击 **压缩** 按钮压缩上下文 |

### 预览与迭代流程

对于 Web 应用开发任务，完成后可在 **浏览器** 窗口中实时预览成果。迭代修改步骤：

1. 点击顶栏中的 **选择元素** 按钮
2. 选中需修改的元素（该元素下方将显示操作项）
3. 选择操作方式：
   - **添加评论到对话**：在输入框中输入修改建议，按回车键，将选中元素和修改建议添加至对话框
   - **添加到对话**：将该元素添加至对话框，然后在对话框中输入修改建议
4. 发送指令，AI 根据新要求进行迭代

### 任务和项目管理

| 编号 | 操作 | 描述 |
|------|------|------|
| 1 | **创建项目** | 点击按钮创建新项目 |
| 2 | **新建任务** | 鼠标悬浮至项目名称处，点击新建任务按钮，在当前项目中新建任务 |
| 3 | **文件管理** | 点击文件管理按钮后，侧栏显示该任务涉及的文件，支持"列表视图"或"树形视图"展示 |

### 网页版与桌面版协同规则

| 协同方向 | 行为 |
|----------|------|
| 桌面版 → 网页版 | 桌面版 Code 模式下发起的云端任务自动同步至网页版，实时更新执行进度 |
| 网页版对同步任务 | 仅支持查看，暂不支持管理 |
| 网页版 → 桌面版 | 网页版发起的任务不会同步至桌面版 |

### 拓展 AI 能力

#### 外部应用授权

可将外部第三方应用集成至 TRAE Work，拓展能力边界。目前支持 GitHub 集成（如 AI 自动创建 PR）。

#### 上下文管理

| 上下文类型 | 说明 |
|------------|------|
| MCP | 通过模型上下文协议（Model Context Protocol），将外部工具和服务连接至 TRAE Work，让 AI 具备调用外部服务的能力 |
| 技能 | 为 AI 提供"专业能力说明书"，使其在执行特定领域任务时表现更出色 |
| 斜杠命令 | 创建快捷指令，用于在对话中快速执行重复性任务 |
| 规则 | 制定代码风格、语言框架等规范，让 AI 输出始终符合个人偏好 |

## 第二部分：移动版快速开始

### 核心定位

TRAE Work 移动版的核心定位是**跨设备的任务下发中心**。采用操作界面与任务执行分离的架构，支持向云端沙箱及已配对的个人电脑下发任务。

### 下载

前往手机应用商店搜索 "TRAE" 应用并下载。

> **iOS 限制**："TRAE" 应用目前尚未在美区 App Store 上架。使用美区 Apple ID 无法搜索并下载，可使用其他地区的 Apple ID 尝试。

### 连接移动版与电脑（完整步骤）

1. 在手机上打开 TRAE Work 移动版，使用手机号登录
2. 在首页左下角，点击任务运行环境 → 显示 **选择任务运行位置** 面板
3. 点击 **连接我的电脑**
4. 在电脑上打开 TRAE Work 桌面版
5. 确保使用与移动版相同的手机号登录桌面版
6. 在桌面版中，前往 **设置** > **TRAE 移动端**，打开 **允许 TRAE 移动端控制当前设备** 开关
7. 在 **仅必要内容会存储在云端** 弹窗中，点击 **确认**

> 隐私说明：为支持跨设备访问，仅对话内容和 AI 生成的产物（如文档、代码变更等）会存储在云端。

8. 根据提示，授权移动版访问电脑中的 `/documents` 目录
   - 连接有效期：**180 天**
   - 连接成功后，电脑出现在 **选择任务运行位置** 列表中，状态为"在线"
9. 按需打开 **"保持电脑处于唤醒状态"** 开关
   - 开启后，TRAE Work 运行期间电脑不会进入睡眠，确保任务顺利完成

### 下发任务（模式 × 运行环境矩阵）

| 模式 | 运行环境 | 操作要求 |
|------|----------|----------|
| Work 模式 | 云端 | 直接在对话框中输入指令并发送 |
| Work 模式 | 电脑 | 先选择存放 AI 产物的文件夹（从已授权目录选择现有文件夹或新建） |
| Code 模式 | 云端 | 先选择代码仓库和分支（需已授权 TRAE Work 使用 GitHub） |
| Code 模式 | 电脑 | 先选择存放 AI 产物的文件夹（从已授权目录选择现有文件夹或新建） |

### 添加上下文（移动版）

1. 在输入框左下角，点击 **+** 按钮
2. 选择上下文类型：

| 类型 | 说明 |
|------|------|
| 相机 | 调用手机相机拍摄照片 |
| 照片 | 从手机相册中选择照片 |
| 文件 | 从手机本地存储中选择文件（**仅 Work 模式支持**） |
| 技能 | 选择在桌面版和网页版中启用的技能 |

### 任务进度监控

| 任务类型 | 同步范围 |
|----------|----------|
| 云端任务 | 移动版、桌面版和网页版实时同步 |
| 本地任务 | 仅在移动版和对应电脑的桌面版之间同步（网页版不支持查看本地任务） |

### 产物预览

移动版支持预览智能体生成的产物（文档、代码、Web 页面等）。

> **限制**：移动版仅支持预览由**云端任务**生成的 Web 页面。本地任务 Web 页面需用桌面版或网页版查看。

### 任务管理

进入方式：对话界面 → 点击左上角 **<** 图标 → 任务中心

| 操作 | 描述 |
|------|------|
| 查看所有任务 | 查看当前账户下的所有任务 |
| 筛选任务 | 通过"全部任务"、"云端"、"{设备名称}"等维度筛选 |
| 进入对话流 | 点击任一任务直接进入该任务的对话流 |
| 新建任务 | 点击右下角 **新建任务** 图标发起全新任务 |

### 个人设置

进入方式：对话界面 → 点击左上角 **<** → 任务中心 → 点击右上角 **个人头像**

| 设置项 | 描述 |
|--------|------|
| 账户 | 开启/关闭隐私模式，或删除账号。隐私模式：启用后，TRAE 不会存储或使用任何聊天交互内容（包括相关代码片段、文件中的内容）用于分析、产品改进或模型训练 |
| 语言 | 切换界面语言，目前支持：英语、简体中文、日语 |
| 通知 | 管理通知权限。开启后在任务完成、中断或需要介入时收到通知 |
| 设备管理 | 查看已连接设备的状态，添加新设备 |
| 三方应用授权 | 管理 TRAE Work 与第三方应用的连接状态（连接/断开）。目前仅支持 GitHub |

### 常见问题

**Q: 移动版 App 进入后台或手机断网，任务会中断吗？**

A: 不会。任务一旦下发，便在指定的运行环境（云端或电脑）中独立执行。App 状态或手机网络状况不影响正在执行的任务。网络恢复后，任务状态和消息会自动同步。

## 关键坑点

- 网页版仅支持查看从桌面版同步的云端任务，暂不支持管理
- 网页版发起的任务不会同步至桌面版
- 网页版仅支持云端运行环境，不支持本地运行环境
- 移动版仅支持预览云端任务的 Web 页面，本地任务 Web 页面需用桌面版/网页版
- 移动版 Code 模式云端任务需先授权 GitHub
- 移动版"文件"上下文仅 Work 模式支持
- iOS 版尚未在美区 App Store 上架
- 移动版与电脑连接有效期 180 天，到期需重新连接

## 可迁移到 AOS 的规则

- 四面板工作区模式（对话/待办/产物/参考）可用于 Agent 任务管理
- 预览→选择元素→评论→迭代的交互模式适用于 AI 辅助开发
- 多端任务同步机制需要统一的状态管理
- 模式 × 运行环境矩阵（Work/Code × 云端/本地）可作为多模式 Agent 系统的权限与路由参考
- 移动版"保持电脑唤醒"开关设计确保长时间任务不中断
- 上下文压缩机制（参考信息面板的压缩按钮）可用于管理长对话的 token 消耗
